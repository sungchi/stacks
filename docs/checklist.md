# Stacks (스택스) Code — 구현·검수 체크리스트

이 문서는 작업 전후에 빠르게 확인하는 실행용 체크리스트다. 제품 규칙의 원본은 [docs/requirements.md](requirements.md)이며, 애셋 정책은 [docs/assets.md](assets.md)를 따른다.

## 1. 작업 시작

- [ ] [AGENTS.md](../AGENTS.md)를 읽었다.
- [ ] 관련 상세 요구사항을 [docs/requirements.md](requirements.md)에서 확인했다.
- [ ] 애셋을 다루면 [docs/assets.md](assets.md)를 확인했다.
- [ ] 이번 작업 범위를 문서, 규칙, UI, 애셋, 테스트 중 무엇으로 볼지 정했다.
- [ ] 요구사항이 부족하면 구현 전에 문서를 갱신했다.
- [ ] `src/`를 수정한다면 마지막에 `npm run build:offline`을 실행할 계획을 세웠다.

## 2. 문서

- [ ] 긴 제품 스펙을 `AGENTS.md`에 넣지 않았다.
- [ ] 새 규칙의 원본은 [docs/requirements.md](requirements.md)에 있다.
- [ ] 체크리스트는 요구사항을 반복 복제하지 않고 실행 항목만 담는다.
- [ ] 새 상세 문서를 추가했다면 [AGENTS.md](../AGENTS.md)의 문서 인덱스를 갱신했다.

## 3. 게임 규칙

- [ ] 캠페인 시작 덱은 0~9 카드 2벌이다.
- [ ] 무한 정원 시작 덱은 일반/특수 능력을 포함한 50장이다.
- [ ] 시작 손패는 5장이다.
- [ ] 정원 더미는 4개다.
- [ ] 기본 목표 평판, 내기, 갈아엎기는 스테이지 설정을 따른다.
- [ ] 같은수, 이웃수, 합 9, 홀짝, 더블 판정이 테스트된다.
- [ ] 이웃수만 콤보 유지로 처리된다.
- [ ] 더미별 추천 결과가 순수 함수로 계산된다.
- [ ] 카드 놓기 후 손패, 더미, 평판, 내기, 메시지가 갱신된다.
- [ ] 손패 보충과 버림더미 재셔플이 동작한다.
- [ ] 갈아엎기 제한과 실패 사유가 동작한다.
- [ ] 목표 평판 달성 시 보상 단계로 간다.
- [ ] 무한 정원은 더미 직접 선택 없이 손패 카드 1장을 자동 배치한다.
- [ ] 무한 정원은 원형 이웃수로 연결을 판정하며 `0-9`, `9-0`도 연결된다.
- [ ] 무한 정원은 가장 낮은 정원 후보를 우선하고, 일반 배치로 높이 차이 2 이상을 만들지 않는다.
- [ ] 무한 정원은 같은 층의 인접 정원 카드도 원형 이웃수 층 줄기로 판정한다.
- [ ] 무한 정원은 배치 시 점수를 올리지 않고, 5장 수확 그룹의 `덧셈 수 합 x 곱셈 계수 합`을 한 번에 더한다.
- [ ] 무한 정원은 10칸이 가득 차도 수확 그룹이 없으면 자동 수확하지 않는다.
- [ ] 무한 정원의 수확/교체 카드는 퇴비 더미로 갔다가 덱이 비면 다시 섞여 돌아온다.

## 4. 성장/저장

- [ ] 발견 보상 3개 후보가 생성된다.
- [ ] 첫 5콤보와 스테이지 클리어가 같은 배치에서 겹쳐도 보상 팝업은 한 번만 열린다.
- [ ] 보상 적용 후 덱, 손패, 콤보 보너스, 도감 상태가 맞게 변한다.
- [ ] 메타 업그레이드 비용, 최대 레벨, 효과가 테스트된다.
- [ ] 캠페인 클리어와 다음 스테이지 해금이 저장된다.
- [ ] 무한 정원 최고 점수, 최고 수확, 생존 턴 기록이 저장된다.
- [ ] 현재 런 저장/복원이 가능하다.
- [ ] 저장 키에 버전이 들어 있다.
- [ ] 소스 변경 후 `public/app.bundle.js`가 갱신됐다.

## 5. UI/UX

- [ ] 첫 화면은 실제 게임 화면이다.
- [ ] 모바일 360px 폭에서 카드, 더미, 버튼, 텍스트가 겹치지 않는다.
- [ ] 데스크톱에서 4정원 보드와 손패가 넓게 읽힌다.
- [ ] `simple/` UI는 큰 패널 없이 상태줄, 라벨을 덜어낸 4정원, 손패, 헤더 메뉴 버튼만 즉시 노출한다.
- [ ] 카드에는 좌상단 합 숫자와 우하단 곱 숫자가 표시되고, `+`와 `x` 보조 기호는 작고 보통 굵기로 읽힌다.
- [ ] `simple/` 보드에 놓인 카드 자체의 좌상단 합 숫자, 우하단 곱 숫자, 컬러 일러스트는 유지된다.
- [ ] `simple/` 모바일은 4정원 2x2와 손패 한 줄이 균형 있게 맞고, 데스크톱은 4정원 1x4와 중앙 정렬 손패가 과하게 퍼지지 않는다.
- [ ] `simple/` 플레이 섹션은 모바일/데스크톱에서 헤더 상단, 보드 중간, 손패 하단의 세로 구획을 유지한다.
- [ ] `simple/` 손패 하단에는 모바일 OS 제스처 영역과 데스크톱 가장자리를 고려한 여유 padding이 있다.
- [ ] `simple/` 메뉴 모달에서 추천, 교체, 새 런과 스테이지, 상점, 도감, 설정에 접근할 수 있다.
- [ ] 메인 UI와 `simple/` UI에서 무한 정원 런을 시작하고 다시 캠페인 스테이지로 돌아갈 수 있다.
- [ ] 무한 정원 손패 카드는 `능력명. 설명` 형식의 고정 능력 박스를 표시한다.
- [ ] 무한 정원 일반 카드는 `일반. 원형 이웃수로 세로/층 줄기를 잇는다.`로 표시한다.
- [ ] 무한 정원 손패 hover/focus 시 자동 배치될 정원, 이유, 예상 수확 후보를 표시한다.
- [ ] 무한 정원에는 남은 덱을 세로 스크롤 목록으로 보는 `덱 보기`가 있다.
- [ ] 추천 표시는 과하지 않고, hover/상세로 이유를 확인할 수 있다.
- [ ] 보상 카드는 박스 전체가 클릭/탭되고, 선택 화면에서 실제 규칙 효과가 먼저 읽힌다.
- [ ] 도감은 다음 발견 목표를 먼저 보여준다.
- [ ] 상점은 구매 전후 최고수 변화를 보여준다.
- [ ] 픽셀 애셋은 흐릿하지 않다.
- [ ] 손패 카드 드래그, 더미 drop highlight, 유효/무효 드롭 피드백이 동작한다.
- [ ] 드래그 중 공중 카드는 transform 기반 이동으로 따라온다.
- [ ] 카드 비행 애니메이션이 끝나기 전 대상 더미에 같은 카드가 미리 보이지 않는다.
- [ ] 손패 카드는 원본 우선순위에 따라 탭/클릭만으로 연결 glow 또는 빈자리 대상에 이동한다.
- [ ] `simple/` 손패 hover/focus 시 해당 카드의 연결 후보와 빈자리 open 후보 강조가 보드 재렌더 없이 표시된다.
- [ ] `simple/` 첫 진입 시 손패 카드가 자동 선택된 것처럼 보이지 않는다.
- [ ] `simple/` 교체는 현재 손패 5장을 모두 바꾸고, 새 손패 카드가 순차 deal 애니메이션으로 들어온다.
- [ ] 애니메이션을 꺼도 상태를 이해할 수 있다.
- [ ] 효과음은 설정에서 끌 수 있고, 꺼도 입력과 상태 변화가 동일하다.
- [ ] 브라우저 콘솔에서 `gardenStacksPerf.summary()`와 `gardenStacksPerf.measureRender(10)`이 측정값을 반환한다.

## 6. 애셋

- [ ] 새 이미지 경로는 `public/assets/garden-stacks/` 아래에 있다.
- [ ] `.DS_Store`, 드래프트, 원본 작업 파일을 불필요하게 추가하지 않았다.
- [ ] 카드 이미지는 `generated/cards/manifest.json`과 맞는다.
- [ ] 종 이미지는 `generated/species/manifest.json`과 맞는다.
- [ ] 누락 이미지 fallback이 있다.

- [ ] 변경 범위에 맞는 자동 테스트 또는 수동 검수 결과를 기록했다.
- [ ] 브라우저에서 기본 화면을 확인했다.
- [ ] `npm run build:offline`이 성공했다.
- [ ] `npm test`가 성공했다.
- [ ] 의도한 파일만 변경했다.
