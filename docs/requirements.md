# Stacks (스택스) Code — 요구사항

이 문서는 Stacks (스택스) 웹 재구현의 상세 요구사항이다. 작업 지침은 [AGENTS.md](../AGENTS.md), 실행 체크리스트는 [docs/checklist.md](checklist.md), 애셋 정책은 [docs/assets.md](assets.md)를 따른다.

> 작업 기준일: 2026-07-05

## 1. 목표

Stacks (스택스) 최신 프로토타입의 `Stacks (스택스) 4.1 / Four Garden Piles`를 `malitmot`식 정적 웹 앱으로 구현한다.

핵심 경험:

- 플레이어는 0~9 정원 카드를 4개의 정원 더미 중 하나에 놓는다.
- 같은수, 이웃수, 합 9, 홀짝 흐름을 이어 더미 배율과 평판을 키운다.
- 제한된 내기와 갈아엎기 안에서 스테이지 목표 평판을 달성한다.
- 런 중 발견 보상과 런 밖 메타 성장을 통해 다음 스테이지를 안정화한다.

만들지 말아야 할 것:

- Lua 프로토타입 전체 이식.
- Lucky Garden, slot, 과거 3.x 히스토리 중심 문서.
- 실제 게임 대신 설명만 있는 랜딩 페이지.
- 카드/더미보다 설명 패널이 먼저 읽히는 UI.

## 2. 용어

| 용어 | 의미 |
|---|---|
| 정원 카드 | 숫자 0~9와 생태 유형/색상/이미지를 가진 카드 |
| 정원 더미 | 카드를 쌓는 4개의 독립 흐름 |
| 평판 | 스테이지 클리어 목표 수치 |
| 내기 | 카드를 한 장 놓는 행동 수 제한 |
| 갈아엎기/교체 | 손패 일부를 버리고 새로 뽑는 제한 행동. 심플 UI에서는 손패 5장을 모두 바꾸는 `교체`로 표기한다. |
| 흐름 | 이전 카드와 새 카드의 숫자 관계 |
| 발견 | 런 중 또는 보상에서 도감에 기록되는 성장 항목 |
| 무한 정원 | 수확 순간에만 점수를 얻으며 최고 점수를 노리는 `endless` 모드 |
| 수확 | 무한 정원에서 원형 이웃수로 이어진 5장 수확 그룹을 제거하고 점수를 한 번에 얻는 행동 |
| 퇴비 더미 | 교체하거나 수확한 카드가 갔다가 덱이 비면 다시 섞여 돌아오는 순환 더미 |

## 3. 구현 방식

- 기본 실행은 정적 웹 앱이다.
- `index.html`, `src/app.js`, `src/styles.css`, `src/game/*.js` 구조를 우선한다.
- Vite/React 같은 프레임워크는 도입하지 않는다.
- 소스는 ES module로 유지하고, `scripts/build-offline.mjs`가 `public/app.bundle.js` classic script 번들을 만든다.
- `index.html`은 직접 열기 호환성을 위해 `public/app.bundle.js`를 로드한다.
- `package.json`은 `type: module`, `test: node --test`, `build:offline`, `open`, `serve` 스크립트를 제공한다.
- 게임 규칙은 `src/game/`의 순수 ES module로 둔다.
- DOM 이벤트, 애니메이션, `localStorage`, Web Share 같은 브라우저 API는 UI 계층에 둔다.

### 4.2 카드 카탈로그

숫자 카드는 다음 속성을 가진다.

| 숫자 | 카드명 | 유형 | 색상 | 대표 이미지 |
|---:|---|---|---|---|
| 0 | 빈 흙 | 지형 | 파랑 | `floor_meadow` |
| 1 | 새싹 | 식물 | 초록 | `occupant_clover` |
| 2 | 꽃망울 | 식물 | 초록 | `occupant_flower` |
| 3 | 벌 | 동물 | 보라 | `occupant_bee` |
| 4 | 물주기 | 관리/도구 | 노랑 | `hand_watering` |
| 5 | 숲그늘 | 지형 | 보라 | `floor_forest_shade_walk` |
| 6 | 나비 | 동물 | 보라 | `bee_tab` |
| 7 | 가지치기 | 관리/도구 | 노랑 | `hand_pruning` |
| 8 | 물가 | 지형 | 파랑 | `floor_wetland` |
| 9 | 철새 | 동물 | 파랑 | `bird_tab` |

유형 순서는 `지형 -> 식물 -> 동물 -> 관리/도구 -> 지형`이다. 색상은 초록, 파랑, 노랑, 보라를 사용한다.

### 4.3 흐름과 점수

이전 더미 맨 위 숫자와 새 숫자의 관계를 판정한다.

- `same`: 같은수.
- `neighbor`: 차이가 1인 이웃수.
- `sum9`: 두 숫자의 합이 9.
- `parity`: 같은 홀짝.
- `double`: 이웃수와 합 9를 동시에 만족하는 경우.
- 첫 카드는 `새 흐름`이다.

콤보 유지 조건은 `neighbor`다. 이 조건을 만족하지 않으면 해당 더미 흐름은 새로 시작한다.

평판 계산 요구:

- 카드 숫자는 기본점이 된다.
- 같은수, 합 9, 홀짝, 지형 선호, 유형 순서, 같은 유형, 같은 색, 색 전환, 4색 완성, 비료, 반짝 카드 보너스를 반영한다.
- 더미의 누적 기본점에 현재 배율을 곱해 획득 평판을 계산한다.
- UI는 최종 평판과 주요 이유를 먼저 보여주고, 상세식은 hover/상세 영역으로 보낸다.

### 4.4 지형

| 지형 | 선호 흐름 | 유형 보너스 |
|---|---|---|
| 초지 | 이웃수 | 식물, 동물, 관리/도구 약보너스 |
| 숲 | 같은수, 홀짝 | 지형, 식물, 관리/도구 약보너스 |
| 습지 | 합 9 | 지형, 동물, 관리/도구 약보너스 |

### 4.5 행동

- 손패 카드를 선택하면 4개 더미별 예상 결과를 계산한다.
- 기본 추천은 가장 좋은 더미를 제시하되, 플레이어가 다른 더미를 선택할 수 있어야 한다.
- 카드를 놓으면 손패에서 제거하고, 내기를 1 줄이고, 평판을 더하고, 해당 더미 상태를 갱신한다.
- 손패가 부족하면 덱에서 채운다. 덱이 비고 버림더미가 있으면 버림더미를 섞어 덱으로 돌린다.
- 갈아엎기는 손패 1장 이상을 버리고 새로 뽑는다. 횟수가 0이면 사용할 수 없다.
- 목표 평판 이상이면 보상 단계로 간다.
- 내기가 0이거나 낼 카드가 없고 목표를 못 채우면 실패한다.

### 4.6 보상과 성장

런 중 첫 5콤보 달성 시 발견 보상을 제공한다. 스테이지 클리어 보상도 발견 풀에서 3개 후보를 제시한다.
같은 카드 배치에서 첫 5콤보와 스테이지 클리어가 동시에 발생하면 스테이지 클리어 보상만 먼저 제공하고, 보상 팝업은 카드 착지 모션이 끝난 뒤 한 번만 열린다.

보상 종류:

- 숫자 카드 추가.
- 0/9처럼 상황에 맞는 숫자 선택 추가.
- 특정 콤보 보너스 증가.
- 카드 색상 접목.
- 카드 유형 접붙이기.
- 덱에서 숫자 제거.
- 보상 선택 UI는 테마명보다 실제 규칙 효과를 먼저 보여준다. 예: `카드 추가`, `합 9 +10%`, `덱 정리`, `색상 접목: 초록`.
- 보상 카드의 시각 박스 전체가 클릭/탭 영역이어야 하며, 이미지나 텍스트만 부분적으로 눌리는 느낌을 주지 않는다.

메타 성장:

- 긴 산책로: 새 런 내기 +1, 최대 2레벨.
- 넓은 모종판: 새 런 갈아엎기 +1, 최대 2레벨.
- 동네 소문: 스테이지 목표 평판 -20, 최대 2레벨.
- 손수레 확장: 손패 +1, 최대 1레벨.
- 새싹 보관함: 시작 덱에 새싹 1 추가, 최대 1레벨.

### 4.7 무한 정원 모드

`endless` 모드는 스테이지 목표 평판과 내기 제한 없이 최고 점수를 노리는 별도 런이다. 상세 설계 배경은 [무한 정원 모드 제안서](analysis/garden-stacks-endless-mode-proposal-2026-07-07.md)와 [자동 배치 연쇄 콤보 모드 제안서](analysis/garden-stacks-auto-chain-combo-proposal-2026-07-08.md)를 기준으로 한다.

MVP 기본값:

- 더미 수: 4
- 시작 덱: 일반 카드와 P0 특수 능력 카드를 합친 총 50장
- 손패: 5장
- 시작 교체: 3
- 더미 용량: 더미마다 일반 카드 10장
- 새 런 셔플: 무한 정원 새 런을 시작할 때마다 새 seed를 만들어 시작 손패와 덱 순서가 반복되지 않게 한다.
- 입력: 플레이어는 손패 카드 1장만 고르고, 더미는 직접 고르지 않는다.
- 자동 배치: 수확 후보를 최우선으로 보고, 그다음 이미 놓인 카드 위에 이어지는 세로 원형 이웃수, 특수 능력, 층 연결, 낮은 정원, 위험도 순으로 결정한다.
- 연결 조건: 원형 이웃수. `0-1`, `1-0`, `8-9`, `9-8`, `0-9`, `9-0`을 연결로 인정한다.
- 층 콤보: 같은 층의 인접 정원 카드도 원형 이웃수면 같은 연결 묶음으로 본다.
- 수확 조건: 새로 놓은 카드가 포함된 세로/층 연결 묶음이 5장 이상이면 5장 수확 그룹을 자동 수확한다.
- 특수 능력: P0에서는 `일반`, `건너잇기`, `접붙이기`, `가지치기`를 사용한다.
- 가지치기: 놓을 수 있는 정원이 있으면 일반 카드처럼 자동 배치한다. 놓을 수 있는 정원이 하나도 없으면 첫 번째 방해카드가 있는 정원의 맨 위 카드 1장을 퇴비로 보내고, 사용한 가지치기 카드도 퇴비로 보낸다.
- 시작 점수: 0

무한 정원 점수 규칙:

- 카드를 놓을 때는 총점이 오르지 않는다.
- 각 카드는 `덧셈 수 = 카드 숫자`, `곱셈 계수 = 9 - 카드 숫자`를 기본값으로 가진다.
- 수확 점수는 수확 그룹 5장의 `덧셈 수 합 x 곱셈 계수 합`이며 수확 시 한 번에 총점에 더한다.
- 더미별 표시용으로 최근 세로 원형 이웃수 줄기의 덧셈 수 합, 곱셈 계수 합, 길이를 기록한다.
- 10칸이 가득 차도 수확 그룹이 없으면 자동 수확하지 않는다.
- 수동 수확 버튼은 없다.
- 수확된 카드와 교체한 카드는 퇴비 더미로 간다.
- 덱이 비면 퇴비 더미를 섞어 새 덱으로 만든다.
- 덱과 퇴비 더미가 모두 비는 예외 상황에서는 기본 50장 무한 정원 덱을 다시 만든다.
- 모든 손패 카드는 카드 안에 고정 능력 박스를 가진다. 표기는 `능력명. 설명` 형식이며 일반 카드는 `일반. 이웃 숫자로 세로/층 줄기를 잇는다.`로 표시한다.
- 손패 hover/focus는 자동 배치 우선순위가 고른 정원 1개만 강조하고, 그 결정 이유와 예상 수확 그룹을 미리 보여준다.
- 무한 정원 모바일/터치 손패에서는 첫 탭으로 해당 카드를 위로 들어 올리고 확대해 같은 미리보기를 보여주며, 같은 카드를 한 번 더 탭하면 자동 배치한다.
- `덱 보기`는 남은 덱 카드를 세로 스크롤 목록으로 보여주며 각 항목의 능력 문구를 함께 표시한다.

무한 정원 실패 조건:

- 손패의 어떤 카드도 자동 배치로 합법 결과를 만들 수 없다.
- 세로 원형 이웃수로 이어지는 배치, 수확 배치, 가지치기 배치는 높이 균형 제한보다 우선한다.
- 세로 연결이 없는 일반/층 배치로 정원 높이 차이가 2 이상 벌어지는 경우는 합법 후보가 아니다.
- 더미가 10칸으로 가득 찬 경우, 이번 배치나 특수 능력으로 수확 그룹을 만들 수 있을 때만 합법 후보가 된다.
- 합법 배치가 없고 교체도 남아 있지 않으면 게임 오버다.

무한 정원 기록:

- 현재 런은 기존 런 저장 키에 저장/복원한다.
- 프로필은 최고 점수, 최고 수확 점수, 최고 생존 턴, 최고 미수확 예상 점수, 플레이 횟수를 저장한다.
- 메인 UI와 `simple/` UI 모두 스테이지 런과 무한 정원 런을 전환할 수 있어야 한다.

## 5. UI/UX

- 첫 화면은 `Stacks (스택스)` 타이틀과 실제 4정원 보드 흔적이 함께 보여야 한다.
- 기본 시각 스타일은 [B/W UI 기본 스타일 가이드](mockups/garden-stacks-ui-2026-07-04/bw-style-guide.md)를 따른다.
- 모바일에서는 한 화면에서 목표, 남은 내기, 더미, 손패, 주요 액션이 겹치지 않아야 한다.
- 데스크톱에서는 4정원 보드와 손패를 넓게 보여주고, 보조 정보는 좌우 레일 또는 hover로 접는다.
- 카드/더미 추천은 낮은 강조로 보여준다. 모든 더미에 과한 점수 마커를 반복하지 않는다.
- 도감은 작은 아이콘 목록보다 다음 발견 목표와 큰 그림을 먼저 보여준다.
- 상점은 구매 결과와 다음 최고수 변화가 즉시 이해되어야 한다.
- 설정, 도감, 상점, 보상은 모달/시트로 열되 현재 보드 맥락을 잃지 않게 한다.
- UI 문법은 흰 바탕, 검은 선, 회색 보조면 중심으로 정리하고, 카드/도감/보상 애셋은 컬러로 유지한다.
- 손패 카드는 `5:7` 비율, 좌상단 합 숫자, 우상단 분류명, 중앙 컬러 일러스트, 우하단 곱 숫자를 기본으로 한다. 합 숫자는 0 외의 숫자 앞에 작은 보통 굵기 `+`를 붙이고, 곱 숫자는 `9 - 숫자` 값이 0이 아닐 때 작은 보통 굵기 `x`를 붙인다. 점수와 보너스 설명은 카드 밖 상세 영역으로 보낸다.
- 무한 정원 손패 카드는 일러스트를 카드 중앙보다 약간 위에 두고, 하단 능력 박스는 `능력명` bold를 가운데 정렬하고 마침표 뒤 짧은 설명은 카드 종류와 같은 폰트 크기로 왼쪽 열맞춤해 최소 2줄까지 읽히도록 충분한 높이를 가진다.
- 선택된 손패 카드는 검게 칠하지 않고 흰 카드가 위로 들린 것처럼 `z-index`, 그림자, 짧은 이동으로 표현한다.
- 픽셀 애셋은 선명하게 렌더링한다. CSS에서 `image-rendering: pixelated`를 사용한다.
- `simple/` 버전은 전체 게임 시스템, 애셋, 저장 키를 공유하면서 메인 화면을 상태줄, 4정원, 손패, 핵심 액션, 메뉴로 축소한다.
- `simple/`은 `?view=mobile` 또는 `?view=desktop` 파라미터로 모바일/데스크톱 레이아웃을 명시할 수 있다. `layout`과 `device`도 같은 값으로 동작한다.
- `simple/` 메인 화면에는 헤더의 작은 `메뉴` 버튼 하나만 노출하고, `추천`, `교체`, `새 런`, 스테이지, 상점, 도감, 설정은 메뉴 모달 안에 둔다.
- `simple/`의 `교체`는 카드별 선택 모드를 열지 않고 현재 손패 5장을 한 번에 모두 교체한다.
- `simple/`의 4정원 보드는 정원명, 점수, 흐름 같은 주변 라벨은 숨기되, 놓인 카드 자체의 좌상단 합 숫자, 우하단 곱 숫자, 컬러 일러스트는 유지한다.
- `simple/`의 모바일과 데스크톱 레이아웃은 모두 4정원을 `1 2 / 3 4` 순서의 2x2 보드로 보여주고, 손패는 중앙 정렬해 과하게 퍼지지 않게 한다.
- `simple/`의 손패는 5장을 작게 눌러 담지 않고 약 1.2배 큰 카드와 약한 부채꼴 배치를 쓴다. 모바일에서는 카드끼리 일부 겹쳐 한 줄에 담으며, 선택/미리보기 중인 카드는 데스크톱 hover처럼 살짝 확대되고 위로 올라온다.
- `simple/`의 플레이 섹션은 헤더를 상단, 손패를 하단, 4정원 보드를 두 영역 사이의 중앙에 두는 세로 구획을 따른다. 작은 화면에서는 잘리지 않도록 패딩과 간격을 줄여 흐른다.
- `simple/`은 iOS/Android 하단 제스처 영역과 데스크톱 가장자리 여백을 고려해 손패 아래에 안정적인 safe-area padding을 둔다.

### 5.1 카드 입력과 모션

기준 출처는 `/Users/sungchi/Desktop/Projects/textbattle2/docs/PRD/emoji-garden/archive/card-interactions-v0.1.md`와 `/Users/sungchi/Desktop/Projects/textbattle2/prototype/emoji_garden/src/scene/garden_stacks_number_play.lua`의 드래그/착지 흐름이다.

- 카드는 `idle -> press -> dragging -> drop_valid/drop_invalid -> placed/shake` 흐름을 따른다.
- 손패 카드는 클릭/키보드 선택과 드래그 앤 드롭을 모두 지원한다.
- 캠페인에서 손패 카드를 탭/클릭하면 원본 Stacks (스택스) 우선순위대로 연결 glow 대상, 빈자리 open 대상 순서로 자동 배치하고 카드 이동, 착지 pulse, 점수 팝을 재생한다.
- 무한 정원에서 데스크톱 손패 클릭 또는 키보드 실행은 더미 직접 선택 없이 자동 배치 우선순위가 고른 정원에 놓고, hover/focus 중에는 그 목적지 1개와 이유만 미리 표시한다. 모바일/터치에서는 첫 탭이 hover/focus와 같은 확대 미리보기이며, 같은 카드 두 번째 탭이 자동 배치 실행이다.
- 카드 이동 애니메이션이 남아 있는 동안 대상 더미는 방금 놓은 카드를 미리 표시하지 않는다. 비행 카드의 CSS 애니메이션 종료 시점과 같은 시점에 더미 카드, 착지 pulse, 점수 pop이 시작된다.
- `simple/`에서는 손패 hover/focus 중 해당 카드의 연결 후보와 빈자리 open 후보를 지연 적용되는 낮은 선 강조로 표시하며, 보드 전체를 다시 렌더하지 않는다.
- 새 런 시작과 전체 손패 교체로 손패가 보충될 때는 새 카드가 순차적으로 들어오는 deal 애니메이션을 재생한다.
- 드래그 중 카드는 공중 카드, 더 커진 그림자, 더미 drop highlight를 보여준다.
- 유효 드롭은 해당 더미에 카드를 놓고 `grid_snap`, 점수 팝, 짧은 착지 pulse를 보여준다.
- 무효 드롭은 손패 카드 `shake`와 짧은 안내를 보여주며 게임 상태를 바꾸지 않는다.
- 개별 모션은 0.4초 안팎으로 짧게 유지하되 프레임 스냅 제한 없이 연속 easing 애니메이션을 우선한다.
- 애니메이션 설정을 끄거나 `prefers-reduced-motion`이면 동일한 입력과 상태 변화가 애니메이션 없이 동작해야 한다.
- 효과음은 별도 오디오 파일 없이 Web Audio 기반 짧은 MIDI식 톤으로 합성한다. 설정에서 끌 수 있으며, 소리가 꺼져도 입력과 상태 변화는 동일해야 한다.

### 5.2 성능과 계측

- 현재 정적 DOM/CSS 구현을 유지하되, 성능 병목은 렌더링, 저장, 액션 처리, 드래그 이동, 규칙 계산 단위로 나누어 측정한다.
- 런타임은 브라우저 콘솔 전용 `window.gardenStacksPerf` API를 제공한다.
- `gardenStacksPerf.summary()`는 `render`, `action`, `keyboard`, `persist`, `game.playCard`, `game.discard`, `game.reward`, `drag.start`, `drag.move` 같은 측정 항목의 횟수, 평균, 최댓값, 마지막 값, 느린 샘플 수를 반환한다.
- `gardenStacksPerf.samples(name, limit)`, `reset()`, `enable(value)`, `measureRender(count)`, `measureNextFrame(name)`으로 최근 샘플과 수동 렌더/프레임 측정을 확인할 수 있어야 한다.
- 측정 샘플은 메모리 증가를 막기 위해 고정 크기 버퍼로 보관한다.
- 드래그 중 공중 카드는 `left/top` 반복 갱신이 아니라 CSS 변수와 `transform: translate3d(...)` 중심으로 이동해 레이아웃 부담을 줄인다.
- 성능 계측은 기본 플레이 결과, 저장 데이터, 보상 선택 결과를 바꾸지 않아야 한다.

## 6. 저장

- 저장은 우선 `localStorage`를 사용한다.
- 저장 키에는 프로젝트명과 버전을 포함한다. 예: `garden-stacks:number:v1:profile`.
- 저장 대상은 메타 재화, 업그레이드, 도감, 캠페인 진행, 현재 런, 설정이다.
- 규칙 버전이 바뀌면 이전 저장을 조용히 무시하거나 명시적으로 마이그레이션한다.

## 7. 애셋

- 런타임 애셋은 `public/assets/garden-stacks/` 아래 파일만 참조한다.
- 앱 런타임 번들은 `public/app.bundle.js`다. 직접 수정하지 말고 `src/`를 수정한 뒤 `npm run build:offline`으로 갱신한다.
- 카드 이미지는 `generated/cards/manifest.json`의 키를 기준으로 매핑한다.
- 종/도감 이미지는 `generated/species/manifest.json`을 기준으로 매핑한다.
- UI 컴포넌트는 `ui/promoted/`, 효과는 `ui/effects/`, 더미 모티프는 `ui/pile_motifs/`를 우선 사용한다.
- 새 애셋을 추가하면 [docs/assets.md](assets.md)에 출처와 사용처를 갱신한다.

## 8. 테스트

자동 테스트 필수 범위:

- 시작 상태 생성.
- 카드 카탈로그 무결성.
- 콤보 판정.
- 더미별 예상 평판.
- 카드 놓기와 손패/덱/버림더미 변화.
- 갈아엎기 제한.
- 스테이지 성공/실패 판정.
- 무한 정원의 원형 이웃수, 자동 균형 배치, 층 콤보, 5장 수확 그룹 점수, 10칸 막힘, 퇴비 순환, 최고 기록 저장.
- 보상 적용.
- 저장/복원.
- 성능 계측 유틸리티의 요약, 샘플 제한, 리셋 동작.

수동 검수 필수 범위:

- 360px, 390px, 768px 이상 폭에서 UI 겹침 없음.
- 첫 화면이 실제 게임 화면으로 읽힘.
- 모든 참조 이미지가 깨지지 않음.
- 키보드와 터치 조작 모두 기본 플레이 가능.
- 브라우저 콘솔에서 `gardenStacksPerf.summary()`와 `gardenStacksPerf.measureRender(10)`이 오류 없이 측정값을 반환함.
