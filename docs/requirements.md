# Stacks (스택스) Code — 요구사항

이 문서는 Stacks (스택스) 웹 재구현의 상세 요구사항이다. 작업 지침은 [AGENTS.md](../AGENTS.md), 실행 체크리스트는 [docs/checklist.md](checklist.md), 애셋 정책은 [docs/assets.md](assets.md)를 따른다.

> 작업 기준일: 2026-07-14

## 1. 목표

Stacks (스택스) 최신 프로토타입의 `Stacks (스택스) 4.1 / Four Garden Piles`를 `malitmot`식 정적 웹 앱으로 구현한다.

핵심 경험:

- 기본 진입은 KST 시간 seed로 생성되는 `매시간 네 정원 수확`이다.
- 플레이어는 손패 5장 중 한 장을 네 정원 중 하나에 직접 드래그하거나 수동 확정한다.
- 한 정원에 네 장이 쌓이면 숫자 합으로 수확하고, 놓은 정원부터 시계방향 꼭대기 숫자가 이어진 길이만큼 곱한다.
- 40장을 모두 사용한 점수를 seed별 최대 목표와 비교해 1~3별 또는 PERFECT를 얻는다.
- 시간이 지나도 현재 게임을 자동 교체하지 않고, 새 게임 준비 알림에서 사용자가 직접 전환한다.
- 기존 무한 정원과 캠페인은 `mode=endless`, `mode=campaign` 명시 주소에서 보존한다.

만들지 말아야 할 것:

- Lua 프로토타입 전체 이식.
- Lucky Garden, slot, 과거 3.x 히스토리 중심 문서.
- 실제 게임 대신 설명만 있는 랜딩 페이지.
- 카드/더미보다 설명 패널이 먼저 읽히는 UI.

## 2. 용어

| 용어 | 의미 |
|---|---|
| 정원 카드 | 숫자 0~9와 생태 유형/색상/이미지를 가진 카드 |
| 정원 더미 | 카드를 쌓는 4개의 독립 흐름 |
| 평판 | 스테이지 클리어 목표 수치 |
| 내기 | 카드를 한 장 놓는 행동 수 제한 |
| 갈아엎기/교체 | 손패 일부를 버리고 새로 뽑는 제한 행동. 심플 UI에서는 손패 5장을 모두 바꾸는 `교체`로 표기한다. |
| 흐름 | 이전 카드와 새 카드의 숫자 관계 |
| 발견 | 런 중 또는 보상에서 도감에 기록되는 성장 항목 |
| 무한 정원 | 수확 순간에만 점수를 얻으며 최고 점수를 노리는 `endless` 모드 |
| 수확 | 무한 정원에서 원형 이웃수로 이어진 5장 수확 그룹을 제거하고 점수를 한 번에 얻는 행동 |
| 퇴비 더미 | 교체하거나 수확한 카드가 갔다가 덱이 비면 다시 섞여 돌아오는 순환 더미 |
| 시간 정원 | KST `YYYYMMDDHH` seed로 한 시간마다 준비되는 기본 40장 점수 게임 |
| 새로받기 | 시간 정원에서 현재 손패를 덱 맨 뒤로 보내고 다음 카드를 받는 게임당 3회의 순환 행동 |
| 연쇄 합 | 한 정원에 쌓여 자동 수확되는 네 숫자의 합 |
| 카드 연쇄 | 수확한 네 카드를 놓인 순서대로 읽어 같은 방향의 `+1` 또는 `-1`이 이어지는 가장 긴 구간. `0↔9`를 인정하고 중복과 방향 반전에서 끊긴다. |
| 정원 연결 | 수확 정원부터 시계방향으로 증가 또는 감소가 이어지는 꼭대기 숫자 흐름 |
| 최장 연쇄 배수 | 카드 연쇄 길이와 정원 연결 길이 중 큰 값으로 정하는 수확의 `×1~×4` 배수 |
| 별 목표 | seed별 solver 목표 점수를 기준으로 계산한 1별, 2별, 3별 점수 |
| PERFECT | 표시된 seed별 최대 목표 점수에 도달한 결과 |

## 3. 구현 방식

- 기본 실행은 정적 웹 앱이다.
- `index.html`, `src/app.js`, `src/styles.css`, `src/game/*.js` 구조를 우선한다.
- Vite/React 같은 프레임워크는 도입하지 않는다.
- 소스는 ES module로 유지하고, `scripts/build-offline.mjs`가 레거시 `public/app.bundle.js`와 기본 시간 게임 `simple/simple.bundle.js` classic script 번들을 만든다.
- 기본 `index.html`은 모드가 없으면 `simple/` 시간 게임으로 이동하고, 명시적 레거시 모드에서는 `public/app.bundle.js`를 로드한다.
- `package.json`은 `type: module`, `test: node --test`, `build:current`, `build:hourly`, `build:legacy`, `build:offline`, `open`, `serve` 스크립트를 제공한다.
- 게임 규칙은 `src/game/`의 순수 ES module로 둔다.
- DOM 이벤트, 애니메이션, `localStorage`, Web Share 같은 브라우저 API는 UI 계층에 둔다.

### 4.2 카드 카탈로그

숫자 카드는 다음 속성을 가진다.

| 숫자 | 카드명 | 유형 | 색상 | 대표 이미지 |
|---:|---|---|---|---|
| 0 | 빈 흙 | 지형 | 파랑 | `floor_meadow` |
| 1 | 새싹 | 식물 | 초록 | `occupant_clover` |
| 2 | 꽃망울 | 식물 | 초록 | `occupant_flower` |
| 3 | 벌 | 동물 | 보라 | `occupant_bee` |
| 4 | 물주기 | 관리/도구 | 노랑 | `hand_watering` |
| 5 | 숲그늘 | 지형 | 보라 | `floor_forest_shade_walk` |
| 6 | 나비 | 동물 | 보라 | `bee_tab` |
| 7 | 가지치기 | 관리/도구 | 노랑 | `hand_pruning` |
| 8 | 물가 | 지형 | 파랑 | `floor_wetland` |
| 9 | 철새 | 동물 | 파랑 | `bird_tab` |

유형 순서는 `지형 -> 식물 -> 동물 -> 관리/도구 -> 지형`이다. 색상은 초록, 파랑, 노랑, 보라를 사용한다.

### 4.3 흐름과 점수

이전 더미 맨 위 숫자와 새 숫자의 관계를 판정한다.

- `same`: 같은수.
- `neighbor`: 차이가 1인 이웃수.
- `sum9`: 두 숫자의 합이 9.
- `parity`: 같은 홀짝.
- `double`: 이웃수와 합 9를 동시에 만족하는 경우.
- 첫 카드는 `새 흐름`이다.

콤보 유지 조건은 `neighbor`다. 이 조건을 만족하지 않으면 해당 더미 흐름은 새로 시작한다.

평판 계산 요구:

- 카드 숫자는 기본점이 된다.
- 같은수, 합 9, 홀짝, 지형 선호, 유형 순서, 같은 유형, 같은 색, 색 전환, 4색 완성, 비료, 반짝 카드 보너스를 반영한다.
- 더미의 누적 기본점에 현재 배율을 곱해 획득 평판을 계산한다.
- UI는 최종 평판과 주요 이유를 먼저 보여주고, 상세식은 hover/상세 영역으로 보낸다.

### 4.5 행동

- 손패 카드를 선택하면 4개 더미별 예상 결과를 계산한다.
- 기본 추천은 가장 좋은 더미를 제시하되, 플레이어가 다른 더미를 선택할 수 있어야 한다.
- 카드를 놓으면 손패에서 제거하고, 내기를 1 줄이고, 평판을 더하고, 해당 더미 상태를 갱신한다.
- 손패가 부족하면 덱에서 채운다. 덱이 비고 버림더미가 있으면 버림더미를 섞어 덱으로 돌린다.
- 갈아엎기는 손패 1장 이상을 버리고 새로 뽑는다. 횟수가 0이면 사용할 수 없다.
- 목표 평판 이상이면 보상 단계로 간다.
- 내기가 0이거나 낼 카드가 없고 목표를 못 채우면 실패한다.

### 4.6 보상과 성장

런 중 첫 5콤보 달성 시 발견 보상을 제공한다. 스테이지 클리어 보상도 발견 풀에서 3개 후보를 제시한다.
같은 카드 배치에서 첫 5콤보와 스테이지 클리어가 동시에 발생하면 스테이지 클리어 보상만 먼저 제공하고, 보상 팝업은 카드 착지 모션이 끝난 뒤 한 번만 열린다.

보상 종류:

- 숫자 카드 추가.
- 0/9처럼 상황에 맞는 숫자 선택 추가.
- 특정 콤보 보너스 증가.
- 카드 색상 접목.
- 카드 유형 접붙이기.
- 덱에서 숫자 제거.
- 보상 선택 UI는 테마명보다 실제 규칙 효과를 먼저 보여준다. 예: `카드 추가`, `합 9 +10%`, `덱 정리`, `색상 접목: 초록`.
- 보상 카드의 시각 박스 전체가 클릭/탭 영역이어야 하며, 이미지나 텍스트만 부분적으로 눌리는 느낌을 주지 않는다.

메타 성장:

- 긴 산책로: 새 런 내기 +1, 최대 2레벨.
- 넓은 모종판: 새 런 갈아엎기 +1, 최대 2레벨.
- 동네 소문: 스테이지 목표 평판 -20, 최대 2레벨.
- 손수레 확장: 손패 +1, 최대 1레벨.
- 새싹 보관함: 시작 덱에 새싹 1 추가, 최대 1레벨.

### 4.7 무한 정원 모드

`endless` 모드는 스테이지 목표 평판과 내기 제한 없이 최고 점수를 노리는 기본 런이다. 주소에 모드를 지정하지 않은 메인 UI와 `simple/` UI는 저장된 무한 정원 런을 이어가며, 저장된 무한 정원 런이 없으면 새 무한 정원을 시작한다. 캠페인은 메뉴 또는 `mode=campaign`으로 명시해 진입한다. 현재 능동형 도구 규칙의 상세 기준은 [능동형 정원 도구 요구사항](requirements/garden-stacks-active-garden-tools-v0.1.md)이며, 설계 배경은 [무한 정원 모드 제안서](analysis/garden-stacks-endless-mode-proposal-2026-07-07.md)와 [자동 배치 연쇄 콤보 모드 제안서](analysis/garden-stacks-auto-chain-combo-proposal-2026-07-08.md)를 참고한다.

MVP 기본값:

- 더미 수: 4
- 시작 덱: 능력 속성이 없는 `0~9` 숫자 카드 각 4장, 총 40장
- 손패: 5장
- 시작 교체: 3
- 더미 용량: 더미마다 일반 카드 10장
- 새 런 셔플: 무한 정원 새 런을 시작할 때마다 새 seed를 만들어 시작 손패와 덱 순서가 반복되지 않게 한다.
- 일반 입력: 플레이어는 손패 카드 1장만 고르고, 더미는 직접 고르지 않는다.
- 자동 배치: 수확 후보를 최우선으로 보고, 그다음 이미 놓인 카드 위에 이어지는 세로 원형 이웃수, 층 연결, 낮은 정원, 위험도 순으로 결정한다.
- 연결 조건: 원형 이웃수. `0-1`, `1-0`, `8-9`, `9-8`, `0-9`, `9-0`을 연결로 인정한다.
- 층 콤보: 같은 층의 인접 정원 카드도 원형 이웃수면 같은 연결 묶음으로 본다.
- 수확 조건: 새로 놓은 카드가 포함된 세로/층 연결 묶음이 5장 이상이면 5장 수확 그룹을 자동 수확한다.
- 도구 보관함: 손패와 분리된 2칸이며 새 런은 `옮겨심기` 1개로 시작한다. 도구는 일회용이고 같은 도구를 중복 보유할 수 있다.
- 도구 보상: 3, 6, 9번째처럼 매 3번째 수확 뒤 서로 다른 도구 2개를 시드 기반으로 제시한다. 빈칸에 보관하거나, 가득 찬 경우 기존 도구를 교체하거나 건너뛴다.
- 옮겨심기: `도구 -> 손패 카드 -> 정원` 순서로 고른 정원에 직접 놓으며 높이 균형과 자동 우선순위를 무시하되 용량 제한은 유지한다.
- 접붙이기: `도구 -> 손패 카드 -> 정원` 순서로 놓고 새 카드와 직교 인접한 비이웃수 카드 1개에 이번 판정만 유효한 임시 연결을 만든다.
- 가지치기: `도구 -> 정원` 순서로 해당 정원의 맨 위 카드 1장을 퇴비로 보낸다. 손패와 점수는 바꾸지 않고 생존 턴은 1 증가한다.
- 시작 점수: 0

무한 정원 점수 규칙:

- 카드를 놓을 때는 총점이 오르지 않는다.
- 각 카드는 `덧셈 수 = 카드 숫자`, `곱셈 계수 = 9 - 카드 숫자`를 기본값으로 가진다.
- 수확 점수는 수확 그룹 5장의 `덧셈 수 합 x 곱셈 계수 합`이며 수확 시 한 번에 총점에 더한다.
- 더미별 표시용으로 최근 세로 원형 이웃수 줄기의 덧셈 수 합, 곱셈 계수 합, 길이를 기록한다.
- 10칸이 가득 차도 수확 그룹이 없으면 자동 수확하지 않는다.
- 수동 수확 버튼은 없다.
- 수확된 카드와 교체한 카드는 퇴비 더미로 간다.
- 덱이 비면 퇴비 더미를 섞어 새 덱으로 만든다.
- 덱과 퇴비 더미가 모두 비는 예외 상황에서는 기본 40장 무한 정원 덱을 다시 만든다.
- 숫자 카드는 카드 안에 능력 박스를 표시하지 않는다. 원형 이웃수 연결은 카드별 능력이 아니라 보드의 고정 규칙이다.
- 손패 hover/focus는 자동 배치 우선순위가 고른 정원 1개만 강조하고, 그 결정 이유와 예상 수확 그룹을 미리 보여준다.
- 무한 정원 손패 카드는 데스크톱 클릭과 모바일/터치 탭 모두 한 번의 입력으로 자동 배치 우선순위가 고른 정원에 놓는다. 데스크톱의 hover/focus는 배치 전에 같은 목적지를 미리 보여주되, 터치에서 별도 확인 탭을 요구하지 않는다.
- 도구가 준비된 동안 일반 자동 배치를 실행하지 않는다. 옮겨심기와 접붙이기는 선택한 카드의 합법 정원 전체를, 가지치기는 카드가 있는 정원과 제거될 맨 위 카드를 미리 표시한다.
- `덱 보기`는 남은 순수 숫자 카드를 세로 스크롤 목록으로 보여준다.

무한 정원 실패 조건:

- 손패의 어떤 카드도 자동 배치로 합법 결과를 만들 수 없고, 보유 도구로도 합법 행동을 만들 수 없다.
- 세로 원형 이웃수로 이어지는 배치와 수확 배치는 높이 균형 제한보다 우선한다.
- 세로 연결이 없는 일반/층 배치로 정원 높이 차이가 2 이상 벌어지는 경우는 합법 후보가 아니다.
- 더미가 10칸으로 가득 찬 경우, 이번 배치로 수확 그룹을 만들 수 있을 때만 합법 후보가 된다. 옮겨심기와 접붙이기도 용량 제한을 무시하지 않는다.
- 합법 배치가 없고 교체도 남아 있지 않으면 게임 오버다.

무한 정원 기록:

- 현재 런은 기존 런 저장 키에 규칙 버전, 보관함, 대기 중 도구 보상, 도구 사용 기록을 포함해 저장/복원한다.
- 능동형 도구 이전 무한 정원 런은 복원하지 않고 새 런을 시작하되 기존 프로필 기록은 유지한다.
- 프로필은 최고 점수, 최고 수확 점수, 최고 생존 턴, 최고 미수확 예상 점수, 플레이 횟수, 전체·도구별 사용 횟수를 저장한다.
- 메인 UI와 `simple/` UI 모두 스테이지 런과 무한 정원 런을 전환할 수 있어야 한다.

### 4.8 매시간 네 정원 수확 기본 모드

상세 설계 근거는 [매시간 네 정원 수확 모드 제안](analysis/garden-stacks-hourly-four-harvest-concept-2026-07-13.md)을 따른다. 기본 주소와 `simple/`은 이 모드를 실행한다.

시간과 seed:

- 기준 시간대는 `Asia/Seoul`이며 seed 형식은 `YYYYMMDDHH`다.
- 실제 카드 셔플에는 seed와 `hourly-four-harvest-v3` 생성기 버전을 함께 사용한다.
- 같은 seed는 시작 손패, 덱 순서, 카드별 `variantId`, 별 목표가 항상 같다.
- 카드 일러스트는 seed, 생성기 버전, 애셋 버전, 카드 ID로 결정하며 숫자별 네 변형을 한 seed 안에서 중복 없이 우선 배정한다.
- 일러스트 변형은 점수와 solver 결과에 영향을 주지 않고, 누락 시 숫자 대표 이미지로 대체한다.

게임 구성:

- 덱은 `0~9` 각 4장, 총 40장이고 손패는 5장이다.
- 정원은 4개이며 2×2 화면 배치는 `A B / D C`, 시계방향 순서는 `A → B → C → D → A`다.
- 플레이어는 손패 카드와 목적지 정원을 모두 직접 확정한다. 자동 목적지는 없다.
- 한 게임에 `새로받기`를 최대 3회 사용할 수 있다. 사용하면 현재 손패를 순서대로 덱 맨 뒤로 보내고 덱 앞에서 최대 5장을 받아 손패를 채우며, 카드를 버리거나 사용한 것으로 세지 않는다.
- 덱에 카드가 없거나 남은 횟수가 0이면 새로받기를 사용할 수 없다. 같은 seed와 같은 행동 순서는 새로받기 결과까지 항상 같다.
- 숫자와 관계없이 모든 카드를 모든 정원에 놓을 수 있다.
- 각 정원은 입력 대기 상태에서 0~3장만 가진다. 네 번째 카드가 들어오면 즉시 수확해 해당 정원만 비운다.
- 덱과 손패 40장을 모두 사용하면 결과 단계로 간다. 남은 정원 카드는 점수화하지 않는다.

점수:

- 기본 수확 점수는 방금 수확한 네 숫자의 합인 `연쇄 합`이다.
- 수확한 네 카드는 놓인 순서 그대로 모든 시작점에서 같은 방향의 `+1` 또는 `-1`이 이어지는 구간을 확인하고 가장 긴 카드 연쇄 길이 `C`를 구한다. `0↔9`를 인정하며 중복, 비연속, 방향 반전에서 해당 구간을 끊는다.
- 수확 카드를 제거하기 전에 방금 놓은 카드 숫자에서 시계방향 다음 정원의 꼭대기 숫자를 확인한다.
- 첫 연결이 `+1`인지 `-1`인지 방향을 고정하고, 같은 방향의 원형 이웃수가 연속되는 동안 연결한다. `0↔9`를 인정한다.
- 빈 정원, 중복 숫자, 비연속 숫자, 방향 반전을 만나면 연결을 멈추며 중간 정원을 건너뛰지 않는다.
- 정원 연결 길이 `G`는 1~4이고, 최종 배수 `M = max(C, G)`다. 여러 카드 연쇄가 같은 길이라도 한 번만 인정하며 카드 연쇄와 정원 연결을 서로 곱하지 않는다.
- 수확 점수는 `네 장 합 × M`이다. `C=1`, `G=1`인 연쇄 없음도 기본 `×1`로 계산하고, 연결된 다른 정원의 카드는 제거하지 않는다.
- 카드에는 별도 곱 계수나 능력 상자가 없다.

목표와 solver:

- 각 seed는 결정적 solver가 만든 플레이 가능 경로와 그 검증 점수를 최대 목표로 저장한다.
- solver 경로는 카드 배치 40수와 선택적으로 사용한 새로받기 최대 3회를 같은 규칙 엔진으로 다시 실행해 표시 점수와 일치해야 한다.
- 1별은 최대 목표의 35%, 2별은 60%, 3별은 80%를 10점 단위로 내린 값이다.
- 최대 목표 이상은 PERFECT다. 플레이 결과는 0~3별과 PERFECT를 지원한다.
- solver 결과는 seed별로 캐시하며 초기 렌더와 드래그 입력을 반복해서 막지 않는다.

시간 전환과 기록:

- 시간이 넘어가도 현재 게임을 자동 종료하거나 새 seed로 교체하지 않는다.
- 타이머가 끝나면 `새 게임 준비됨`과 시작 버튼을 보여주고, 사용자가 눌렀을 때만 최신 seed로 전환한다.
- `다시하기`는 게임방법 `?` 모달 안에서만 제공한다. 같은 seed의 덱, 손패, 새로받기 3회를 초기화하되 최고 점수, 최고 별, PERFECT, 도전 횟수는 유지한다.
- 브라우저 새로고침은 진행 중이거나 완료된 활성 seed를 복원하고 최신 seed는 대기 상태로 둔다.
- 저장 키는 `garden-stacks:hourly-v3:<seed>:run`, `:best`, `:solution`과 활성 seed 키를 사용한다. v2 런과 solver 캐시는 새 점수 규칙에 사용하지 않는다.
- 공유는 선택 언어의 `Stacks #시드 별 점수 / 최대 목표 URL` 한 줄이며 모바일은 Web Share, 데스크톱은 클립보드 복사와 토스트를 사용한다.

## 5. UI/UX

- 첫 화면은 `Stacks (스택스)` 타이틀과 실제 4정원 보드 흔적이 함께 보여야 한다.
- 첫 방문은 `malitmot`처럼 실제 게임 보드 위에 게임방법 모달을 자동으로 연다. 이 최초 모달에는 언어 선택과 계속하기를 제공하되 아직 플레이를 시작하기 전이므로 다시하기를 표시하지 않는다. 닫은 뒤에는 확인 상태를 저장하고, 이후 `?`로 연 모달에서만 현재 게임 다시하기를 제공한다.
- 기본 시각 스타일은 [B/W UI 기본 스타일 가이드](mockups/garden-stacks-ui-2026-07-04/bw-style-guide.md)를 따른다.
- 모바일에서는 한 화면에서 목표, 남은 내기, 더미, 손패, 주요 액션이 겹치지 않아야 한다.
- 데스크톱에서는 4정원 보드와 손패를 넓게 보여주고, 보조 정보는 좌우 레일 또는 hover로 접는다.
- 카드/더미 추천은 낮은 강조로 보여준다. 모든 더미에 과한 점수 마커를 반복하지 않는다.
- 도감은 작은 아이콘 목록보다 다음 발견 목표와 큰 그림을 먼저 보여준다.
- 상점은 구매 결과와 다음 최고수 변화가 즉시 이해되어야 한다.
- 설정, 도감, 상점, 보상은 모달/시트로 열되 현재 보드 맥락을 잃지 않게 한다.
- UI 문법은 흰 바탕, 검은 선, 회색 보조면 중심으로 정리하고, 카드/도감/보상 애셋은 컬러로 유지한다.
- 캠페인과 무한 정원 카드 표면은 `5:7` 비율, 좌상단 합 숫자, 중앙 컬러 일러스트, 일러스트 아래의 큰 카드명, 우하단 곱 숫자를 유지한다. 시간 정원 카드는 숫자 하나, seed 일러스트, 카드명만 표시하고 `+`, `x`, 능력 상자를 사용하지 않는다.
- 무한 정원 손패 카드는 능력 박스 없이 합 숫자, 기존 대비 약 1.2배 큰 일러스트, 굵고 큰 카드명, 곱 숫자가 먼저 읽혀야 한다.
- 무한 정원 도구 보관함은 손패와 분리한 고정 크기 정사각형 2칸으로 표시한다. 각 보유 슬롯은 일러스트, 도구명, `1회`를 보여주고 빈 슬롯은 낮은 대비로 표시한다.
- 준비된 도구는 외곽선, 짧은 상승, 설명 한 줄, 대상 강조로 표시하며 색 없이도 준비·합법 대상·위험·취소 상태를 구분할 수 있어야 한다.
- 도구 보상은 수확 모션이 끝난 뒤 한 번만 열리고 카드 전체 영역으로 선택한다. 보관함이 가득 찬 경우 교체 대상 2칸과 건너뛰기를 제공한다.
- 선택된 손패 카드는 검게 칠하지 않고 흰 카드가 위로 들린 것처럼 `z-index`, 그림자, 짧은 이동으로 표현한다.
- 픽셀 애셋은 선명하게 렌더링한다. CSS에서 `image-rendering: pixelated`를 사용한다.
- `simple/`은 기본 시간 정원의 실제 플레이 화면이며 별 목표, 4정원, 손패, 강조된 새로받기, 게임방법, 공유, 새 시간 전환만 노출한다. 상단의 `점수 · 별 · 수확 · 최고` 요약 영역은 표시하지 않고, 다시하기는 게임방법 모달 안에 둔다.
- `simple/`의 모든 사용자 노출 문구, 접근성 이름, 카드명, 결과 공유 문구, 문서 제목과 설명은 한국어·영어·일본어를 지원한다. 별도 국제화 라이브러리 없이 카드 ID와 평면 번역 사전을 사용하며 게임 상태와 seed 결정성은 언어와 무관하게 유지한다.
- 언어는 저장된 최근 선택을 먼저 적용하고, 저장값이 없으면 `navigator.languages`와 `navigator.language`의 첫 지원 언어(`ko`, `en`, `ja`)를 감지하며 그 외 언어는 한국어로 되돌린다. 게임방법 모달에는 세 언어의 자기 언어명 라디오 버튼을 제공하고 선택 즉시 전체 화면과 `html[lang]`을 갱신한다.
- `simple/`의 카드 새로받기는 손패 아래에 충분한 위 간격을 두고 전체 손패 폭의 가로 중앙에 고정한다. 흰 배경, 2px 검은 테두리, 다른 주요 버튼과 같은 모서리 굴곡, 단단한 오프셋 그림자와 눌림 이동을 사용하는 절제된 네오브루탈리즘 버튼으로 표시한다. 크기가 다른 라벨·남은 횟수·분모는 같은 세로 중심축에 정렬하며, 카드 사용량은 버튼 정렬에 영향을 주지 않는 오른쪽 보조 정보로 둔다.
- `simple/`은 `?view=mobile` 또는 `?view=desktop` 파라미터로 모바일/데스크톱 레이아웃을 명시할 수 있다. 모드 파라미터가 없거나 `mode=hourly`면 시간 정원으로 진입한다.
- 루트의 `mode=endless`와 `mode=campaign`은 기존 레거시 UI를 직접 연다.
- `simple/`의 4정원은 각각 고정된 네 카드 슬롯과 `n/4`, 목적지별 예상 합·카드 연쇄·정원 연결 중 최장 배수와 점수를 표시한다.
- `simple/`의 모바일과 데스크톱 레이아웃은 모두 4정원을 `A B / D C` 순서의 2x2 보드로 보여주고, 손패는 중앙 정렬해 과하게 퍼지지 않게 한다. 상단 순서 표시는 `A → B → C → D`로 읽힌다.
- `simple/`의 손패는 5장을 작게 눌러 담지 않고 약 1.2배 큰 카드와 약한 부채꼴 배치를 쓴다. 모바일에서는 카드끼리 일부 겹쳐 한 줄에 담으며, 선택/미리보기 중인 카드는 데스크톱 hover처럼 살짝 확대되고 위로 올라온다.
- `simple/` 모바일은 헤더, 4정원, 손패의 세로 흐름을 사용하고 데스크톱은 헤더 아래 보드와 손패를 좌우로 배치한다. 작은 화면에서는 잘리지 않도록 패딩과 간격을 줄인다.
- `simple/`은 iOS/Android 하단 제스처 영역과 데스크톱 가장자리 여백을 고려해 손패 아래에 안정적인 safe-area padding을 둔다.

### 5.1 카드 입력과 모션

기준 출처는 `/Users/sungchi/Desktop/Projects/textbattle2/docs/PRD/emoji-garden/archive/card-interactions-v0.1.md`와 `/Users/sungchi/Desktop/Projects/textbattle2/prototype/emoji_garden/src/scene/garden_stacks_number_play.lua`의 드래그/착지 흐름이다.

- 카드는 `idle -> press -> dragging -> drop_valid/drop_invalid -> placed/shake` 흐름을 따른다.
- 손패 카드는 클릭/키보드 선택과 드래그 앤 드롭을 모두 지원한다.
- 시간 정원은 자동 목적지를 사용하지 않는다. 드래그를 기본 입력으로 하고 카드 탭 후 정원 탭, `1~5` 후 `Q/W/E/R`을 동등한 수동 확정 입력으로 지원한다.
- 손패 카드를 누르면 선택 상승 상태를 즉시 보여주되 이동 임계값을 넘기기 전에는 탭으로 처리한다. 탭은 해당 카드를 선택하거나 이미 선택된 카드를 해제하고, 임계값을 넘긴 뒤에는 선택 상승 상태와 공중 카드를 유지한 채 드래그하며, 취소 입력은 누르기 전 선택으로 복원한다.
- 마우스나 트랙패드처럼 hover 가능한 정밀 포인터에서는 카드를 한 번 클릭해 손을 떼도 공중 카드가 커서를 계속 따라오는 `클릭해서 들기` 상태로 전환한다. 버튼을 누른 채 유지할 필요가 없으며, 정원을 다시 클릭하면 현재 공중 위치에서 스냅 배치한다. 같은 카드 재클릭, 빈 영역 클릭, `Escape`, 창 포커스 이탈은 이 상태를 취소한다. 터치에서는 공중 카드를 띄우지 않고 카드 탭 후 정원 탭 방식을 유지한다.
- 시간 정원 드래그 중 네 정원은 모두 유효 대상으로 표시하고 각 목적지의 `n/4`, 네 장 합, 최장 연쇄 배수, 최종 예상 점수를 보여준다.
- 캠페인에서 손패 카드를 탭/클릭하면 원본 Stacks (스택스) 우선순위대로 연결 glow 대상, 빈자리 open 대상 순서로 자동 배치하고 카드 이동, 착지 pulse, 점수 팝을 재생한다.
- 무한 정원에서 손패 클릭·탭 또는 키보드 실행은 더미 직접 선택 없이 한 번의 입력으로 자동 배치 우선순위가 고른 정원에 놓는다. 데스크톱 hover/focus 중에는 그 목적지 1개와 이유만 미리 표시한다.
- 시간 정원 드래그 중 공중 카드는 포인터를 보간하거나 늦게 따라오는 이동 애니메이션 없이 같은 프레임의 위치에 표시한다.
- 유효 드롭은 놓은 위치에서 대상 슬롯까지 0.18초 이하로 짧게 스냅한 뒤 보드 카드, 착지 pulse, 효과음을 같은 시점에 시작한다. 스냅 중 대상 더미는 방금 놓은 카드를 미리 표시하지 않는다.
- 카드 스냅과 수확 모션 중에는 다음 카드 선택, 정원 확정, 새로받기를 차단한다.
- 시간 정원 키보드는 `1~5`로 손패를 고르고 `Q/W/E/R`로 정원을 확정하며 `Escape`로 선택을 취소한다. 레거시 무한 정원의 `6/7` 도구 입력은 명시 모드에서 보존한다.
- `simple/`에서는 손패 hover/focus 중 해당 카드가 실제로 자동 배치될 우선 정원 1개만 지연 적용되는 낮은 선 강조로 표시하며, 보드 전체를 다시 렌더하지 않는다.
- 손패 미리보기 전환 전후에 각 정원 셀과 2x2 보드의 크기·위치는 변하지 않는다. 미리보기 문구 공간은 항상 예약하고 추천 표시는 레이아웃에 영향을 주지 않는 테두리 또는 그림자 애니메이션으로 표현한다.
- 메인 UI의 손패 hover/focus도 기존 DOM의 카드·정원 강조와 미리보기만 갱신한다. 포인터 진입·이탈 때문에 전체 플레이 DOM을 반복 생성해서는 안 된다.
- 새 런 시작과 새로받기로 손패가 바뀔 때는 새 카드가 촤라락 순차적으로 들어오는 deal 애니메이션과 Web Audio 효과음을 함께 재생한다.
- 네 번째 카드로 수확하면 수확 정원 중심에서 `+획득점수`가 약 1초 동안 위로 날아가며 사라지고, `malitmot` 정답 성공 효과와 같은 흑백 확산 링, 6개 스파크, 짧은 보드 팝을 함께 재생한다. 마지막 카드 수확도 이 효과를 끝까지 보여준 뒤 결과 모달을 연다.
- 드래그 중 카드는 공중 카드, 더 커진 그림자, 더미 drop highlight를 보여준다.
- 드래그 시작 시 안정적인 루트 요소가 포인터를 캡처하고, 터치는 10px·펜은 8px·마우스는 6px를 넘겨야 드래그로 전환한다.
- `pointercancel`과 포인터 캡처 상실은 게임 상태를 바꾸지 않고 공중 카드와 drop highlight만 정리한다.
- 유효 드롭은 해당 더미에 카드를 놓고 `grid_snap`, 짧은 착지 pulse와 효과음을 보여준다.
- 무효 드롭은 손패 카드 `shake`와 짧은 안내를 보여주며 게임 상태를 바꾸지 않는다.
- 개별 모션은 0.4초 안팎으로 짧게 유지하되 프레임 스냅 제한 없이 연속 easing 애니메이션을 우선한다.
- 애니메이션 설정을 끄거나 `prefers-reduced-motion`이면 동일한 입력과 상태 변화가 애니메이션 없이 동작해야 한다.
- 효과음은 별도 오디오 파일 없이 Web Audio 기반 짧은 MIDI식 톤으로 합성한다. Web Audio가 사용자 입력 뒤에도 재개되지 않는 브라우저에서는 같은 음표를 코드에서 PCM WAV로 만들어 준비된 `HTMLAudioElement`로 재생한다. 설정에서 끌 수 있으며, 소리가 꺼져도 입력과 상태 변화는 동일해야 한다.
- Web Audio 컨텍스트는 첫 포인터 또는 키보드 입력 안에서 미리 생성·재개해, 비동기 스냅이나 수확 종료 뒤 재생하는 효과음도 브라우저 자동재생 정책에 막히지 않게 한다.

### 5.2 성능과 계측

- 현재 정적 DOM/CSS 구현을 유지하되, 성능 병목은 렌더링, 저장, 액션 처리, 드래그 이동, 규칙 계산 단위로 나누어 측정한다.
- 런타임은 브라우저 콘솔 전용 `window.gardenStacksPerf` API를 제공한다.
- `gardenStacksPerf.summary()`는 `render`, `action`, `keyboard`, `persist`, `game.playCard`, `game.discard`, `game.reward`, `drag.start`, `drag.move` 같은 측정 항목의 횟수, 평균, 최댓값, 마지막 값, 느린 샘플 수를 반환한다.
- `gardenStacksPerf.samples(name, limit)`, `reset()`, `enable(value)`, `measureRender(count)`, `measureNextFrame(name)`으로 최근 샘플과 수동 렌더/프레임 측정을 확인할 수 있어야 한다.
- 측정 샘플은 메모리 증가를 막기 위해 고정 크기 버퍼로 보관한다.
- 드래그 중 공중 카드는 `left/top` 반복 갱신이 아니라 CSS 변수와 `transform: translate3d(...)` 중심으로 이동해 레이아웃 부담을 줄인다.
- 연속 `pointermove`는 `requestAnimationFrame`당 한 번으로 합치고 `drag.start`, `drag.move`, `drag.end`를 별도로 측정한다.
- 성능 계측은 기본 플레이 결과, 저장 데이터, 보상 선택 결과를 바꾸지 않아야 한다.

### 5.3 빌드 대상

- `npm run build:current`와 `npm run build:hourly`는 현재 기본 시간 정원의 `simple/simple.bundle.js`만 생성한다.
- `npm run build:legacy`는 명시 모드용 `public/app.bundle.js`만 생성한다.
- `npm run build:offline`은 두 대상을 모두 생성하며 전체 완료 검수에 사용한다.

## 6. 저장

- 저장은 우선 `localStorage`를 사용한다.
- 저장 키에는 프로젝트명과 버전을 포함한다. 예: `garden-stacks:number:v1:profile`.
- 저장 대상은 메타 재화, 업그레이드, 도감, 캠페인 진행, 현재 런, 설정이다.
- 시간 정원은 활성 seed, seed별 런, 최고 기록, solver 목표를 `garden-stacks:hourly-v1` 네임스페이스에 분리해 저장한다.
- 최근 언어는 `garden-stacks:language:v1`, 최초 게임방법 확인 여부는 `garden-stacks:hourly:help-seen:v1`에 저장한다. 저장소를 사용할 수 없어도 현재 탭에서는 감지 언어와 게임 플레이가 동작해야 한다.
- 규칙 버전이 바뀌면 이전 저장을 조용히 무시하거나 명시적으로 마이그레이션한다.

## 7. 애셋

- 런타임 애셋은 `public/assets/garden-stacks/` 아래 파일만 참조한다.
- 런타임 번들은 레거시 `public/app.bundle.js`와 기본 시간 게임 `simple/simple.bundle.js`다. 직접 수정하지 않는다. 시간 정원만 바꾸면 `npm run build:current`, 레거시만 바꾸면 `npm run build:legacy`, 전체 완료 검수에서는 `npm run build:offline`으로 갱신한다.
- 카드 이미지는 `generated/cards/manifest.json`의 키를 기준으로 매핑한다.
- 종/도감 이미지는 `generated/species/manifest.json`을 기준으로 매핑한다.
- UI 컴포넌트는 `ui/promoted/`, 효과는 `ui/effects/`, 더미 모티프는 `ui/pile_motifs/`를 우선 사용한다.
- 새 애셋을 추가하면 [docs/assets.md](assets.md)에 출처와 사용처를 갱신한다.
