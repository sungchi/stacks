# Stacks (스택스) 능동형 정원 도구 요구사항 v0.1

> 작성일: 2026-07-11
> 상태: 구현 기준. [docs/requirements.md](../requirements.md) 4.7의 세부 규칙이다.
> 대상: 무한 정원 `endless` 모드
> 배경: 숫자 카드에 결합된 자동 발동 특수능력을 분리해 희소한 능동 도구와 직접적인 전략 선택으로 전환한다.

## 1. 목적

현재 무한 정원의 `건너잇기`, `접붙이기`, `가지치기`는 숫자와 점수 값을 가진 일반 카드에 능력 속성으로 결합되어 있다. 모든 카드는 같은 손패, 같은 자동 배치, 같은 수확 점수 구조를 사용하므로 특수능력이 별도 선택이라기보다 숫자 카드의 수동적 보너스로 읽힌다.

이 요구사항의 목적은 다음과 같다.

- 숫자 카드와 특수능력의 역할을 분리한다.
- 특수능력을 희소한 일회용 자원으로 만들어 사용 시점에 의미를 준다.
- 기본 자동 배치는 유지하되, 도구를 사용할 때만 위치·연결·공간에 직접 개입하게 한다.
- 일반 카드의 정보량을 줄이고 도구의 시각적 특별함을 강화한다.
- 현재 과도하게 안정적인 장기 생존 압력을 함께 조정한다.

## 2. 범위

P0 범위:

- 무한 정원 숫자 덱을 순수 숫자 카드로 재구성한다.
- 숫자 카드 밖에 최대 2칸의 `정원 도구` 보관함을 추가한다.
- `옮겨심기`, `접붙이기`, `가지치기` 3종을 구현한다.
- 수확 3회 보상을 교체 회복에서 도구 선택으로 변경한다.
- 도구 선택, 사용, 취소, 저장, 복원, 결과 기록을 지원한다.
- 기존 런타임 일러스트를 재사용한다.

P0에서 제외:

- `되새김`, `비료` 및 추가 도구.
- 캠페인 모드의 도구 시스템.
- 도구 강화, 영구 업그레이드, 희귀도.
- 신규 이미지 제작.
- 일일 시드와 온라인 순위표.

## 3. 용어

| 용어 | 의미 |
|---|---|
| 숫자 카드 | `0~9` 숫자, 합 숫자, 곱 숫자, 일러스트만 가진 기본 플레이 카드 |
| 정원 도구 | 숫자 카드와 분리된 일회용 능동 특수능력 |
| 도구 보관함 | 현재 런에서 보유한 도구를 최대 2개 저장하는 UI와 상태 |
| 준비된 도구 | 플레이어가 선택했지만 아직 성공적으로 사용하지 않은 도구 |
| 도구 대상 | 도구 효과가 적용될 손패 카드, 정원 또는 연결선 |
| 도구 보상 | 매 3번째 수확 뒤 제시되는 도구 선택 후보 |

## 4. 핵심 구조

### 4.1 숫자 덱

- 시작 숫자 덱은 `0~9` 각 4장, 총 40장이다.
- 모든 숫자 카드는 `ability = normal` 같은 개별 능력 값을 갖지 않는다.
- 숫자 카드는 기존처럼 `덧셈 수 = 숫자`, `곱셈 계수 = 9 - 숫자`를 가진다.
- 숫자 카드의 연결, 수확, 퇴비 순환 규칙은 현재 무한 정원 규칙을 유지한다.
- 숫자 카드에는 `일반. 이웃 숫자로 세로/층 줄기를 잇는다.` 능력 박스를 표시하지 않는다.
- 일반 연결 규칙은 보드의 고정 규칙이며 카드별 능력으로 취급하지 않는다.

### 4.2 도구 보관함

- 도구 보관함의 최대 크기는 2칸이다.
- 새 런은 `옮겨심기` 1개를 가진 상태로 시작한다.
- 같은 도구를 중복 보유할 수 있다. 각 항목은 1회 사용 단위다.
- 도구는 손패 크기를 차지하지 않는다.
- 도구는 숫자 덱, 손패, 퇴비 더미에 들어가지 않는다.
- 성공적으로 사용한 도구는 런에서 제거된다.
- 도구를 선택하거나 취소하는 것만으로는 소모되지 않는다.
- 유효하지 않은 대상, 취소, 애니메이션 중 입력으로는 소모되지 않는다.

### 4.3 도구 획득

- 현재 `수확 3회마다 교체 +1` 규칙을 제거한다.
- 3, 6, 9번째처럼 매 3번째 수확이 끝나면 도구 보상을 1회 연다.
- 도구 보상은 서로 다른 도구 2개를 시드 기반으로 제시한다.
- 보상 UI는 수확 카드 제거와 점수 모션이 끝난 뒤 정확히 한 번 열린다.
- 보관함에 빈칸이 있으면 선택한 도구를 빈칸에 넣는다.
- 보관함이 가득 찼으면 기존 도구 1개를 교체하거나 보상을 건너뛸 수 있다.
- 보상 후보와 선택 결과는 현재 런 저장에 포함한다.
- 도구 보상은 교체 횟수를 회복하지 않는다.

## 5. P0 도구 규칙

### 5.1 옮겨심기

- 내부 ID: `relocate`
- 역할: 위치 조작.
- 입력 순서: `도구 선택 -> 손패 카드 선택 -> 정원 선택`.
- 선택한 숫자 카드를 플레이어가 고른 정원에 놓는다.
- 이번 배치는 정원 높이 균형 제한과 자동 배치 우선순위를 무시한다.
- 정원 용량 제한은 무시하지 않는다. 정원이 가득 찼다면 이번 배치로 수확이 발생할 때만 합법이다.
- 배치 뒤 숫자 연결과 수확 판정은 일반 규칙을 그대로 사용한다.
- 연결되지 않는 위치를 선택할 수 있으며, 그 위험은 실행 전에 미리 표시한다.
- 성공적으로 카드가 놓인 순간 도구를 소모한다.

전략적 질문:

- 자동 배치가 고르지 않은 정원에 새 줄기를 시작할 것인가.
- 당장 연결되는 위치보다 다음 층 수확을 준비할 위치를 선택할 것인가.

### 5.2 접붙이기

- 내부 ID: `graft`
- 역할: 숫자 관계 조작.
- 입력 순서: `도구 선택 -> 손패 카드 선택 -> 정원 선택`.
- 선택한 숫자 카드를 선택한 정원에 놓고, 새 카드와 직교로 인접한 비이웃수 카드 1개에 임시 연결선을 만든다.
- 직교 인접은 같은 정원의 바로 아래 카드 또는 같은 층의 좌우 정원 카드다.
- 임시 연결 후보가 여러 개면 연결 묶음 크기, 예상 수확 점수, 정원 순서 기준으로 하나를 결정한다.
- 임시 연결선은 이번 배치의 연결 묶음과 수확 판정에만 사용하고 상태에 영구 저장하지 않는다.
- 유효한 비이웃수 연결 후보가 없으면 해당 정원은 도구 사용 대상으로 표시하지 않는다.
- 성공적으로 임시 연결을 만든 순간 도구를 소모한다.

전략적 질문:

- 작은 줄기 두 개를 지금 합쳐 수확할 것인가.
- 더 높은 점수 조합을 기다리지 않고 공간을 먼저 비울 것인가.

### 5.3 가지치기

- 내부 ID: `prune`
- 역할: 공간 조작.
- 입력 순서: `도구 선택 -> 정원 선택`.
- 선택한 정원의 맨 위 숫자 카드 1장을 퇴비 더미로 보낸다.
- 손패 카드를 사용하거나 보충하지 않는다.
- 점수를 얻거나 수확 횟수를 올리지 않는다.
- 비어 있는 정원은 대상으로 선택할 수 없다.
- 제거 후 해당 정원의 연결 합, 곱, 길이를 다시 계산한다.
- 도구 사용은 생존 턴 `turnCount`를 1 올린다.
- 제거와 재계산이 성공한 순간 도구를 소모한다.

전략적 질문:

- 막힘 직전까지 보관할 것인가.
- 잘못 놓인 카드 하나를 미리 제거해 큰 수확을 준비할 것인가.

## 6. 입력과 상태 전이

### 6.1 기본 입력

- 도구를 사용하지 않을 때는 현재 무한 정원처럼 손패 카드 1장을 선택하고 자동 배치한다.
- 도구 버튼을 누르면 해당 도구를 `준비됨` 상태로 바꾼다.
- 준비된 도구를 다시 누르거나 취소 버튼, `Escape`를 누르면 도구 선택을 취소한다.
- 다른 도구를 누르면 준비된 도구만 교체되며 어떤 도구도 소모하지 않는다.
- 도구가 준비된 동안 일반 자동 배치를 즉시 실행하지 않는다.

### 6.2 실행 순서

`옮겨심기`와 `접붙이기`:

1. 도구를 선택한다.
2. 손패 카드 1장을 선택한다.
3. 보드에 도구 적용 후 합법인 정원만 강조한다.
4. 정원 1개를 선택하면 카드 이동과 도구 효과를 실행한다.
5. 카드 착지 뒤 연결, 수확, 점수, 손패 보충을 처리한다.
6. 성공한 경우 도구를 제거하고 선택 상태를 초기화한다.

`가지치기`:

1. 도구를 선택한다.
2. 카드가 있는 정원만 강조한다.
3. 정원 1개를 선택하면 맨 위 카드를 퇴비로 이동한다.
4. 정원 상태와 합법 배치를 다시 계산한다.
5. 성공한 경우 도구를 제거하고 선택 상태를 초기화한다.

### 6.3 입력 차단

- 카드 비행, 수확, deal, 도구 보상 모션 중에는 새 도구를 준비하거나 실행할 수 없다.
- 보상, 결과, 게임 오버 단계에서는 도구를 사용할 수 없다.
- 잘못된 대상 입력은 상태를 바꾸지 않고 짧은 거절 피드백만 표시한다.
- 도구 실행과 수확 보상이 같은 행동에서 발생하면 수확 모션 뒤 도구 보상을 한 번만 연다.

## 7. UI/UX 요구사항

### 7.1 숫자 카드

- 숫자 카드는 현재 `5:7` 비율, 좌상단 합 숫자, 약 1.2배 큰 중앙 일러스트, 일러스트 아래의 큰 카드명, 우하단 곱 숫자를 유지한다.
- 일반 능력 박스를 제거해 일러스트와 숫자 조합을 우선 읽게 한다.
- 도구 사용 중 선택된 숫자 카드는 흰 카드가 위로 들린 기존 선택 표현을 유지한다.

### 7.2 도구 보관함

- 도구 보관함은 손패와 시각적으로 분리된 2개의 고정 크기 정사각형 슬롯이다.
- 슬롯은 손패 위 또는 옆에 두되 360px 화면에서 손패와 겹치지 않아야 한다.
- 도구에는 숫자, 합 숫자, 곱 숫자를 표시하지 않는다.
- 각 슬롯은 일러스트, 도구명, 남은 사용 단위만 표시한다.
- 빈 슬롯은 장식 카드처럼 보이지 않는 낮은 대비의 빈칸으로 표시한다.
- 준비된 도구는 검은 외곽선, 짧은 상승, 대상 강조로 표현한다.
- 도구 설명은 hover/focus 또는 선택 중 상세 한 줄로 제공한다.

### 7.3 대상 미리보기

- 일반 플레이는 실제 자동 배치 목적지 1개만 강조한다.
- `옮겨심기`는 합법 정원 전체와 각 정원의 연결 여부를 표시한다.
- `접붙이기`는 합법 정원과 새로 생길 임시 연결선 1개를 표시한다.
- `가지치기`는 제거될 맨 위 카드와 제거 후 정원 높이를 표시한다.
- 위험한 결과는 색만으로 구분하지 않고 `연결 끊김`, `수확`, `1칸 확보` 같은 짧은 텍스트를 함께 제공한다.

### 7.4 접근성과 모바일

- 도구 슬롯은 `button`으로 구현하고 도구명, 효과, 상태를 접근성 이름에 포함한다.
- 키보드는 도구 선택, 손패 선택, 정원 선택, 취소를 모두 지원한다.
- 모바일에서는 도구 선택 뒤 카드 또는 정원을 한 번 더 눌러 확정하는 의도적 다단계 입력을 사용한다.
- 도구 선택 상태가 화면 밖으로 밀리지 않도록 360px와 safe-area에서 검수한다.
- `prefers-reduced-motion`에서는 연결선과 이동 모션 없이 같은 상태 변화와 설명을 제공한다.

- 일러스트는 비율을 유지하고 `object-fit: contain`을 사용한다.
- 접붙이기와 가지치기 일러스트가 실제 슬롯 크기에서 혼동되는지 44px, 56px, 72px 표시 크기로 확인한다.
- 두 도구가 혼동되면 신규 애셋을 바로 만들지 않고 접붙이기의 기존 대체 후보 `hand_watering.png`를 먼저 비교 검수한다.
- 애셋 경로가 없을 때는 `card_locked_unknown.png`를 사용하고 콘솔 경고를 남긴다.

## 9. 상태와 저장

무한 정원 런 상태에 최소 다음 필드를 추가한다.

```js
{
  tools: [{ id: "relocate" }, { id: "prune" }],
  armedToolIndex: null,
  pendingToolReward: null,
  toolRewardCount: 0,
  toolsUsed: 0,
  toolHistory: []
}
```

- `armedToolIndex`는 저장하지 않거나 복원 시 `null`로 초기화한다.
- 보상 선택 중 새로고침하면 같은 후보와 보관함 상태를 복원한다.
- `toolHistory`는 도구 ID, 대상 정원, 사용한 카드 ID, 수확 여부, 점수 변화를 기록한다.
- 프로필에는 전체 도구 사용 횟수와 도구별 사용 횟수를 저장할 수 있다.
- 규칙 구조가 크게 달라지므로 기존 무한 정원 현재 런은 복원하지 않고 새 런을 시작한다.
- 기존 최고 점수, 최고 수확, 생존 턴 같은 프로필 기록은 유지한다.

## 10. 밸런스 원칙

- 도구는 좋은 숫자 카드가 아니라 제한된 규칙 개입권이다.
- 도구 사용만으로 직접 점수를 만들지 않는다.
- 접붙이기로 수확해도 점수는 수확된 숫자 카드 5장의 값으로만 계산한다.
- 옮겨심기와 접붙이기는 용량 제한을 무시하지 않는다.
- 가지치기는 공간을 만들지만 점수, 수확 횟수, 교체 횟수를 주지 않는다.
- 보관함 2칸은 `지금 사용하거나 다음 보상을 포기할지` 선택하게 만드는 상한이다.
- 수확 3회 보상을 교체 회복에서 도구로 바꿔 장기 생존 자원의 자동 재생산을 줄인다.

초기 밸런스 검증:

- 기존 내장 추천을 사용한 시드 20개의 100턴 생존 결과와 새 규칙 결과를 비교한다.
- 새 규칙에서도 20개 시드가 모두 100턴을 생존하면 도구 지급 주기보다 교체 수 또는 정원 용량을 먼저 재검토한다.
- 특정 도구가 전체 사용의 60%를 넘으면 다른 도구의 발동 조건과 결과 가치를 비교한다.

## 11. 자동 테스트

- 숫자 시작 덱이 `0~9` 각 4장, 총 40장이고 능력 속성을 갖지 않는다.
- 새 런이 옮겨심기 1개와 빈 도구 슬롯 1개로 시작한다.
- 도구 보관함은 2개를 넘지 않는다.
- 매 3번째 수확이 교체를 회복하지 않고 도구 보상을 만든다.
- 도구 보상 후보는 같은 시드와 상태에서 동일하다.
- 취소, 잘못된 대상, 입력 차단 상태에서 도구가 소모되지 않는다.
- 옮겨심기는 자동 배치와 높이 제한을 무시하지만 용량 제한은 유지한다.
- 접붙이기는 비이웃수 임시 연결 1개만 만들고 다음 행동에 남기지 않는다.
- 가지치기는 맨 위 카드 1장만 퇴비로 보내고 손패와 점수를 바꾸지 않는다.
- 도구는 숫자 덱과 퇴비 순환에 들어가지 않는다.
- 도구 사용으로 수확하면 카드 비행, 수확, 보상 순서가 한 번씩만 처리된다.
- 도구 보상 중 저장·복원해도 후보와 보관함 상태가 유지된다.
- 기존 프로필 기록을 유지하면서 구버전 무한 정원 런만 새 런으로 교체한다.

## 12. 수동 검수

- 360px, 390px, 768px 이상에서 도구 슬롯, 손패, 보드가 겹치지 않는다.
- 일반 숫자 카드에서 능력 박스가 제거되어 일러스트와 두 숫자가 먼저 읽힌다.
- 세 도구가 숫자 카드와 다른 UI 요소로 즉시 구분된다.
- 옮겨심기, 접붙이기, 가지치기가 각각 위치, 연결, 공간 조작으로 다르게 보인다.
- 준비, 대상 선택, 취소, 실행, 소모 상태를 색 없이도 이해할 수 있다.
- 카드 비행이 끝나기 전 보드에 카드가 미리 나타나지 않는다.
- 수확 모션이 끝나기 전 도구 보상이 열리지 않는다.
- 키보드와 터치로 세 도구를 모두 사용할 수 있다.
- 기존 세 일러스트가 실제 슬롯 크기에서 서로 구분된다.

## 13. 완료 조건

- 숫자 카드와 도구가 데이터, 덱, UI, 저장 구조에서 분리되어 있다.
- 플레이어는 일반 턴에는 카드만 고르고, 도구 턴에는 도구 사용 시점과 대상을 직접 고른다.
- 세 도구가 동일한 자동 연결 보너스로 수렴하지 않는다.
- 도구 사용 전 결과와 소모 여부를 미리 확인할 수 있다.
- 새 애셋 제작 없이 기존 런타임 일러스트로 P0 UI가 완성된다.
- 관련 자동 테스트, 모바일/데스크톱 수동 검수, `npm test`, `npm run build:offline`이 성공한다.
