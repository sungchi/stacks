# Stacks (스택스) B/W UI 기본 스타일 가이드

> 작성일: 2026-07-05  
> 적용 범위: Stacks (스택스) 기본 UI 스타일, [B/W 목업](bw-index.html), [mockups-bw.css](mockups-bw.css)
> 기준 감각: malitmot식 흰 바탕, 검은 선, 회색 보조면, 강한 타이포

## 핵심 원칙

**UI는 흑백으로 조용하게, 게임 애셋은 컬러로 선명하게 둔다.**

Stacks (스택스)의 기본 시각 스타일은 B/W 스타일이다. 모든 것을 흑백화하는 방향이 아니라, 버튼, 패널, 상태, 선택 표시 같은 UI 문법은 흑백으로 정리하고, 카드 일러스트와 발견/보상 애셋은 컬러로 남겨 플레이 대상이 먼저 보이게 한다.

컬러 목업은 비교용 참고안으로 유지한다. 구현 판단이 갈릴 때는 이 문서를 우선한다.

## 팔레트

- 배경: `#ffffff`
- 본문/강조선: `#111111`
- 보조 텍스트: `#6f6f6f`
- 얇은 구분선: `#d9d9d9`
- 보조 면: `#f5f5f5`

새로운 강조색은 추가하지 않는다. 상태 차이는 색보다 선 두께, 채움, 그림자, 패턴, 위치로 구분한다.

## 카드

- 손패 카드는 `5:7` 비율을 우선한다.
- 카드 상단은 `숫자 + 분류명`만 둔다.
- 중앙은 컬러 일러스트만 둔다.
- 카드 하단 점수, 긴 이름, 보너스 설명은 카드 밖 상세 영역으로 보낸다.
- 선택된 카드는 검게 칠하지 않는다. 흰 카드가 위로 들린 것처럼 `translateY`, `z-index`, 그림자로 표현한다.

## 버튼

- 기본 버튼은 흰 배경, 얇은 회색 보더, 굵은 텍스트다.
- 주 행동 버튼만 검은 배경과 흰 텍스트를 쓴다.
- `추천 놓기` 같은 보조 행동은 flat ghost button으로 둔다.
- 버튼에 불필요한 입체 효과를 넣지 않는다. 입체감은 주로 선택된 카드에만 쓴다.

## 보드와 더미

- 보드는 밝은 회색 면 위에 흰 더미 카드를 놓는다.
- 더미 기본선은 검은 2px 선으로 명확히 한다.
- 추천/드롭 대상은 굵은 검은 테두리나 안쪽 선으로 표시한다.
- 흐름이 끊기는 더미는 붉은색 대신 dashed 선이나 낮은 대비 회색을 쓴다.
- 빈 더미는 사선 패턴이나 점선으로 “비어 있음”을 보여준다.

## 애셋

- 카드, 도감, 보상 이미지는 `filter: none`으로 컬러를 유지한다.
- UI 색상과 게임 오브젝트 색상이 경쟁하지 않게, 주변 UI는 흑백/회색으로 낮춘다.
- 애셋을 설명 텍스트로 반복하지 않는다. 카드 안에서는 이미지가 주인공이다.

## 정보 위계

- 첫 번째로 보여야 하는 것: 손패 카드와 4개 더미.
- 두 번째로 보여야 하는 것: 현재 목표, 남은 내기, 추천 한 줄.
- 세 번째로 보여야 하는 것: 점수 이유, 보너스 상세, 상점/도감 메타 정보.

모바일에서는 특히 상단 설명이 보드를 밀어내지 않게 한다. 플레이 중 가장 큰 물체는 메뉴가 아니라 카드와 더미여야 한다.

## 하지 않을 것

- 전체 애셋을 흑백 필터로 바꾸지 않는다.
- 선택된 카드를 검은 카드로 뒤집힌 것처럼 만들지 않는다.
- 추천 버튼에 별도의 그림자나 장식 효과를 주지 않는다.
- 카드 안에 점수, 설명, 보너스 문장을 다시 넣지 않는다.
- 보조 메뉴가 플레이 액션보다 더 강하게 보이게 하지 않는다.
