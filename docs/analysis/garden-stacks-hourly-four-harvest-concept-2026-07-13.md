# Stacks 매시간 네 정원 수확 모드 제안

> 작성일: 2026-07-13
>
> 개정: 2026-07-16, v0.7
>
> 기준 화면: 최신 `simple/` 모바일·데스크톱 무한 정원
>
> 참고 구현: 로컬 `malitmot`의 KST 시간 시드, 수동 새 게임 전환, 시드별 저장, 결과 공유 흐름
>
> 상태: 기존 수확 솔리테어와 별도로 검증할 신규 시간별 점수 모드 제안

> 현재 구현 메모: v0.4에서는 별도였던 카드 연쇄와 정원 연결을 `수확 카드 4장 → 시계방향의 다른 정원 꼭대기`로 이어지는 하나의 수확 경로로 통합했다. v0.5에서는 고정 별 목표를 `250·400·600점`으로 상향했다. v0.6에서는 다른 정원까지 이어진 전체 연쇄 길이를 잘라내지 않고 최대 `×7`까지 적용한다. v0.7에서는 생물군 분류만으로 고정 900점에 도달하는 지배 전략을 완화하기 위해 같은 생물군 보너스를 `×5`에서 `×3`으로 낮췄다. 아래 v0.3의 두 후보 비교와 solver 최대점수 설명은 설계 이력이며, 현재 규칙은 [제품 요구사항 4.8](../requirements.md#48-매시간-네-정원-수확-기본-모드)을 우선한다.

## 1. 한 줄 결론

**이 방향이 기존 수확 솔리테어보다 단순하고 반복 도전에 강하다.**

권장 규칙은 다음 한 문장으로 정리된다.

> **손패 한 장을 네 정원 중 하나에 직접 놓는다. 네 장이 쌓이면 합으로 수확하고, 수확 카드에서 시계방향 정원 꼭대기까지 이어지는 하나의 연쇄 또는 같은 생물군 네 장 중 더 높은 배수 하나를 곱한다.**

한 판은 시간 seed로 정해진 40장을 모두 놓으면 끝난다. 모든 시간은 같은 `250·400·600점` 목표를 사용하며 시작 시 solver를 실행하지 않는다. 숫자가 맞지 않아도 모든 카드는 놓을 수 있으므로 합법 수 부족으로 막히지 않는다. 실패는 `카드를 못 놓음`이 아니라 별 목표에 도달하지 못한 결과다.

이 모드에서는 자동 목적지를 제거하고 손패 카드의 수동 드래그를 기본 입력으로 삼는 것이 맞다. 핵심 전략이 `어느 정원 꼭대기를 어떤 숫자로 유지할 것인가`이므로 시스템이 목적지를 대신 고르면 게임의 중심 판단이 사라진다.

## 2. 기존 제안보다 강해지는 이유

기존 수확 솔리테어 v0.2는 다음 선택을 제공했다.

- 윗묶음 옮기기
- 손패 카드 선택
- 정원 선택
- 5장 수확

전략은 생기지만 턴 단계와 묶음 선택 규칙을 설명해야 했다. 새 제안은 윗묶음 이동을 없애고 카드 배치 하나에 두 종류의 목표를 겹친다.

- 한 정원에는 어떤 숫자든 네 장을 모아 합 수확을 만든다.
- 수확한 네 장의 마지막 숫자에서 시계방향 꼭대기 숫자까지 하나의 연쇄를 만든다.

플레이어는 드래그 한 번만 하지만, 놓을 곳을 고를 때 현재 정원의 네 장 합과 수확 카드에서 다른 정원까지 이어질 최장 연쇄를 함께 비교한다. 행동은 더 단순하고 판단은 더 넓어진다.

고정 별 목표도 같은 seed를 반복하면서 `이번에는 3별 경로를 찾는다`는 분명한 재도전 이유를 만든다. v0.5의 `×4` 상한 기준 72개 시간 seed 표본에서 무작위 플레이 중앙값은 267점, 단순 연쇄 우선 플레이 중앙값은 436점이었다. 넓은 beam 탐색으로 낮은 후보를 재검증했을 때 모두 600점 이상 경로가 확인되어 `250·400·600점`을 각각 기본 이해, 전략적 플레이, 고득점 도전 기준으로 채택했다. v0.6의 최대 `×7` 규칙은 이 분포보다 점수를 높일 수 있으므로 목표는 비교를 위해 우선 유지하고 후속 표본에서 다시 검증한다.

## 3. 20초 규칙

1. 손패 5장 중 한 장을 네 정원 중 하나로 드래그한다.
2. 한 정원에 네 장이 쌓이면 숫자와 관계없이 자동 수확하고 네 장의 합을 구한다.
3. 수확한 네 장에서 시계방향의 다른 정원 꼭대기까지 한 줄로 읽고, 가장 긴 연속 숫자의 전체 길이를 합에 곱한다(최대 ×7).
4. 시간 seed로 정해진 카드 40장을 모두 사용하면 게임이 끝난다.
5. 표시된 점수에 따라 1별, 2별, 3별로 클리어한다.

핵심 문장:

> **네 장은 합, 연쇄는 ×1~×7, 같은 생물군 네 장은 ×3.**

## 4. 카드와 보드

### 4.1 덱과 손패

- 덱은 `0~9` 각 4장, 총 40장이다.
- 시작 손패는 5장이다.
- 카드를 놓으면 덱에서 1장을 보충한다.
- 덱이 비면 남은 손패를 모두 사용할 때까지 진행한다.
- 셔플 순서는 KST 시간 seed와 생성기 버전으로 결정한다.
- 같은 시간 seed의 재도전은 항상 같은 시작 손패와 카드 순서를 사용한다.

카드에는 숫자 하나만 있다. 기존 우하단 곱 계수와 카드별 점수 계수는 제거한다.

### 4.2 시간 seed 동식물과 생물군

- 기존 종 애셋에서 식물 20종과 동물 20종, 총 40종을 사용하며 한 덱 안에서 그림을 중복하지 않는다.
- 지형과 관리/도구 이미지는 시간 정원 덱에 넣지 않는다.
- 40종은 `꽃·나무·양서류·새·곤충` 5개 `comboTypeId` 생물군으로 나뉘며, 각 생물군은 서로 다른 동식물 여덟 장을 가진다.
- 각 생물군의 여덟 장에는 서로 다른 숫자 여덟 개를 배정한다.
- 숫자 0~9도 각각 네 장을 유지하므로 총 40장이다.
- 같은 seed를 다시 시작하거나 저장 상태를 복원하면 같은 생물군 순서, 숫자 배정, 카드 순서와 일러스트가 나온다.
- 시간이 바뀌면 생물군 순서, 숫자와 생물군 안의 일러스트 배정이 달라질 수 있다.
- 종 애셋이 누락되면 중립 이미지로 대체하되 `speciesId`와 `comboTypeId`는 유지한다.

권장 결정 키:

```text
<YYYYMMDDHH>:<generatorVersion>:<assetVersion>:<cardInstanceId>
```

`speciesId`는 실제 동식물과 일러스트를 식별한다. 같은 생물군 수확과 solver에는 별도 `comboTypeId`를 사용한다. 이미지 파일 자체에는 별도 점수 속성이 없으며 solver는 숫자, 생물군, 카드 순서를 함께 사용한다.

### 4.3 네 정원

- 정원은 최신 심플 UI처럼 4개를 `2×2`로 보여준다.
- 각 정원은 네 번째 카드가 들어오는 즉시 수확되므로, 입력 대기 상태에서는 0~3장만 가진다.
- 숫자와 관계없이 모든 손패 카드를 모든 정원에 놓을 수 있다.
- 네 정원은 항상 유효한 드롭 대상이다.

정원이 가득 차서 막히는 상태 자체가 없다. 네 장째가 들어오면 그 정원은 즉시 비워지므로 운 나쁜 손패나 과거 적재 때문에 입력이 중단되지 않는다. 대신 카드 놓인 순서의 연쇄와 네 정원의 연결 중 어느 쪽으로 더 긴 흐름을 만들었는지가 점수를 가른다.

## 5. 합 수확과 정원 연결

### 5.1 네 장 합 수확

한 정원에 네 번째 카드가 놓이면 네 숫자의 관계와 무관하게 자동 수확한다.

기본 점수:

```text
연쇄 합 = 정원에 쌓인 네 숫자의 합
```

예시:

- `2, 8, 4, 5` 수확: `2+8+4+5 = 19점`
- `8, 9, 0, 1` 수확: `8+9+0+1 = 18점`

네 장은 수확 후 보드에서 제거되고 해당 정원은 빈 정원이 된다. 40장을 모두 한 정원에만 놓더라도 10번 수확할 수 있지만, 높은 점수를 얻으려면 한 정원 안의 카드 순서 또는 다른 정원의 꼭대기 숫자로 긴 연쇄를 만들어야 한다.

### 5.2 카드 연쇄 판정

수확한 네 카드는 놓인 순서 그대로 읽는다. 모든 카드 위치를 시작점으로 `+1` 또는 `-1`이 같은 방향으로 이어지는 구간을 찾고 가장 긴 하나만 인정한다. `0↔9`는 이어지며 중복, 비연속, 방향 반전에서 끊긴다.

- `1, 2, 7, 8`: 길이 2 연쇄가 둘이지만 최장 길이는 `C=2`
- `2, 2, 3, 4`: 최장 구간 `2→3→4`, `C=3`
- `9, 0, 1, 2`: 원형 증가 연쇄, `C=4`
- `2, 2, 2, 2`: 연쇄 없음, `C=1`

### 5.3 시계방향 순서

정원은 숫자 카드와 혼동되지 않도록 `A B / D C`로 표시하고, 화면상 시계방향은 다음 순서다.

```text
A → B
↑   ↓
D ← C
```

따라서 전체 순환은 `A → B → C → D → A`다. 어느 정원에서 수확하든 그 정원을 시작점으로 같은 방향을 따른다.

### 5.4 정원 연결 판정

네 번째 카드를 놓아 수확이 발생한 순간, 카드를 제거하기 전 꼭대기 숫자를 판정한다.

1. 방금 카드를 놓은 정원의 꼭대기 숫자에서 시작한다.
2. 시계방향 다음 정원의 꼭대기 숫자가 `+1` 또는 `-1`이면 연결한다.
3. 첫 연결이 증가인지 감소인지 방향을 고정한다.
4. 같은 방향으로 이어지는 동안 다음 정원을 계속 연결한다.
5. 빈 정원, 중복 숫자, 방향 반전, 비연속 숫자를 만나면 즉시 멈춘다.
6. 중간 정원을 건너뛰고 그다음 정원과 연결할 수는 없다.

`0`과 `9`는 원형으로 이어진다. 정원 연결 길이 `G`는 최소 1, 최대 4다. `G=1`이면 추가 정원 연결이 없는 상태다.

### 5.5 최종 점수

```text
같은 생물군 후보 S = 네 장의 comboTypeId가 모두 같으면 3, 아니면 1
최종 배수 M = max(최장 카드 연쇄 C, 정원 연결 G, 같은 생물군 S)
수확 점수 = 네 장 합 × M
```

배수 후보를 다시 곱하거나 더하지 않는다. 카드 연쇄와 정원 연결이 모두 없고 같은 생물군도 아니면 `M=1`이므로 네 장 합을 그대로 얻는다. 서로 다른 동식물 네 장의 `comboTypeId`가 같으면 같은 생물군 `×3` 후보를 얻는다.

연결된 다른 정원의 꼭대기 숫자는 네 장 합에 다시 더하지 않고, 정원 연결 후보 `G`만 결정한다.

예시 1: 연결 없음

- 정원 A의 네 장: `2, 8, 4, 5`, 네 장 합 `19`
- 최장 카드 연쇄 `4→5`, `C=2`
- 정원 A의 놓은 카드 `5`, 시계방향 정원 B의 꼭대기 `2`
- `5 → 2`는 이어지지 않으므로 `G=1`
- `M=max(2,1)=2`, 점수: `19×2 = 38점`

예시 2: 세 정원 증가 연결

- 정원 A의 네 장: `2, 8, 4, 5`, 네 장 합 `19`, `C=2`
- 수확 직전 꼭대기: 정원 A=`5`, 정원 B=`6`, 정원 C=`7`, 정원 D=`2`
- 시계방향으로 `5 → 6 → 7`까지 이어져 `G=3`
- `M=max(2,3)=3`, 점수: `19×3 = 57점`

예시 3: 네 정원 원형 감소 연결

- 수확 정원 꼭대기부터 시계방향으로 `1 → 0 → 9 → 8`
- `G=4`
- 카드 연쇄가 `C≤4`이고 네 장 합이 `18`이면 `M=4`, 점수: `18×4 = 72점`

예시 4: 같은 생물군 네 장

- 한 정원의 네 카드가 모두 `꽃` 생물군이고 실제 동식물 그림은 서로 다르며 숫자 합이 `19`
- 카드 연쇄가 `C=2`, 정원 연결이 `G=1`이면 같은 생물군 후보 `S=3`
- `M=max(2,1,3)=3`, 점수: `19×3 = 57점`

연결에 사용된 다른 정원의 꼭대기 카드는 제거하지 않는다. 그 정원들은 다음 수확의 연결 고리 또는 새로운 시작점으로 계속 남는다.

### 5.6 이 공식이 명확한 이유

- 한 정원에 쌓은 네 장의 가치는 모두 더한다.
- 카드 놓인 순서, 정원 사이 연결, 같은 생물군 조합 중 가장 높은 하나가 배수가 된다.
- 같은 생물군 후보는 `×3`, 수확 연쇄는 최대 `×7`로 두고 후보를 겹치지 않는다.
- 카드에는 별도 곱 계수가 필요 없다.
- `0`이 포함되어도 곱 전체가 0점이 되지 않는다.
- 연결된 다른 정원의 카드를 제거하지 않아 보드 상태를 쉽게 추적할 수 있다.

화면에는 일반 수확은 **합 N × 최장 M = 점수**, 같은 생물군 후보가 이기면 **합 N × 같은 생물군 3 = 점수**로 보여준다. 게임방법에서는 세 후보 중 가장 높은 하나만 쓴다고 설명한다.

## 6. 자동 수확 순서

1. 손패 카드를 정원에 놓는다.
2. 정원이 1~3장이면 수확 없이 손패를 보충한다.
3. 네 번째 카드라면 제거 전 상태에서 네 장 합과 최장 카드 연쇄 `C`를 계산한다.
4. 방금 놓은 정원부터 시계방향 정원 연결 길이 `G`를 계산한다.
5. 네 장의 `comboTypeId`가 모두 같은지 확인해 같은 생물군 후보 `S`를 계산한다.
6. `M=max(C,G,S)`와 `네 장 합 × M`을 총점에 더한다.
7. 방금 놓은 정원의 네 장만 제거한다.
8. 수확 모션이 끝난 뒤 손패를 보충한다.

한 번의 배치로 수확 정원은 정확히 하나이므로 동시 수확 우선순위가 필요 없다. 드래그 중에는 일반 결과를 `합 19 × 최장 4 = 76`, 같은 생물군 결과를 `합 19 × 같은 생물군 3 = 57`처럼 미리 보여준다.

## 7. 점수와 별 목표

### 7.1 최대 점수

각 시간 seed에 대해 모든 합법적인 `손패 카드 선택 × 정원 선택` 경로를 탐색해 이론상 최대 점수 `M`을 계산한다.

최대 점수와 함께 최적 행동 경로 하나를 저장한다. 저장한 경로를 규칙 엔진으로 다시 재생해 실제로 `M`이 나오는지 검증해야 한다.

정확한 최대 점수가 없으면 별 목표가 시드마다 불공정해질 수 있으므로 추정값이나 휴리스틱 봇 최고 점수를 `M`으로 표시해서는 안 된다.

### 7.2 별 컷

기본 비율:

| 결과 | 초기 기준 |
|---|---:|
| PERFECT | `M` |
| 3별 | `M`의 80% |
| 2별 | `M`의 60% |
| 1별 | `M`의 35% |

별 점수는 보기 좋은 10점 단위로 내림한다.

```text
3별 = floor(M × 0.80 / 10) × 10
2별 = floor(M × 0.60 / 10) × 10
1별 = floor(M × 0.35 / 10) × 10
```

예시: 최대 점수 `M = 347`

- PERFECT: `347`
- 3별: `270`
- 2별: `200`
- 1별: `120`

3별을 최대 점수와 같게 두지는 않는다. 최적해 재현만 3별이 되면 대부분의 플레이어가 목표를 포기한다. 최대 점수 달성은 별과 별도의 `PERFECT` 기록으로 남긴다.

### 7.3 seed 품질 보정

단순 비율만 적용한 뒤 자동 플레이 분포로 난이도를 검증한다.

목표 구간:

- 1별 도달: 첫 도전 플레이의 80~95%
- 2별 도달: 45~70%
- 3별 도달: 10~25%
- PERFECT: 해법 탐색 또는 반복 학습이 필요한 희귀 기록

별 간 점수 차가 너무 작거나, 무작위 배치만으로 3별이 자주 나오는 seed는 폐기하고 시간 seed salt를 바꿔 다시 생성한다. 별 컷을 임의로 크게 흔드는 것보다 퍼즐 자체를 다시 만드는 편이 공통 도전의 신뢰를 지킨다.

### 7.4 계산 위치

최대 점수 완전 탐색을 브라우저 메인 스레드에서 매시간 실행하면 첫 화면과 입력이 느려질 수 있다. 권장 방식은 배포 전에 미래 시간 seed의 다음 정보를 미리 생성하는 것이다.

- 생성기 버전
- 시간 seed
- 덱 순서 검증 해시
- 최대 점수
- 1~3별 컷
- 최적 경로 검증 해시

런타임은 시간 seed로 덱을 재현하고 작은 시간별 메타데이터만 읽는다. 메타데이터가 없는 시간에는 검증된 기본 challenge를 제공하며, 불완전한 최대 점수를 표시하지 않는다.

solver는 손패를 정렬된 집합으로 캐시한다. 네 정원은 시계방향 상대 순서를 보존해야 하므로 임의로 정렬하지 않고, 회전·반사해도 같은 상태만 대칭 정규화한다. 각 정원이 최대 3장만 유지하므로 기존 10장 스택이나 윗묶음 이동 규칙보다 탐색 상태가 작다. P0에서는 먼저 1,000개 seed의 계산 시간과 상태 수를 측정해 40장 완전 탐색이 현실적인지 확인해야 한다.

## 8. 말잇못식 시간별 게임 흐름

로컬 `malitmot` 구현에서 가져올 핵심은 자동 교체가 아니라 **새 게임 준비 상태**다.

### 8.1 시간 seed

- 기준 시간대: `Asia/Seoul`
- seed 형식: `YYYYMMDDHH`
- 실제 RNG 입력: `seed + 생성기 버전`
- 같은 seed는 덱, 최대 점수, 별 컷이 항상 같다.

### 8.2 시간이 바뀔 때

- 진행 중인 게임을 자동 종료하거나 새 seed로 교체하지 않는다.
- 타이머 `mm:ss`가 끝나면 `새 게임 · 준비됨`으로 바뀐다.
- 현재 게임 근처에 `새 게임이 준비됐어요` 배너와 `시작` 버튼을 보여준다.
- 플레이어가 `시작`을 눌렀을 때만 최신 seed로 전환한다.
- 현재 seed의 진행과 최고 결과는 seed별 저장 키에 남긴다.

### 8.3 다시하기

- `다시하기`는 현재 seed의 보드와 손패를 처음 상태로 되돌린다.
- 같은 seed의 최고 점수, 최고 별, PERFECT 여부는 유지한다.
- 주 플레이 화면에서는 새로받기를 더 강한 액션으로 보여주고, 다시하기는 `?` 게임방법 모달 안에서만 제공한다.
- 새 게임이 준비된 뒤에도 `현재 판 다시하기`와 `새 게임 시작`을 함께 제공한다.
- 브라우저 새로고침은 진행 중인 현재 seed를 복원하며 자동으로 최신 seed로 바꾸지 않는다.

권장 저장 키:

```text
garden-stacks:hourly-v6:<seed>:run
garden-stacks:hourly-v6:<seed>:best
garden-stacks:hourly-v6:<seed>:solution
```

## 9. 말잇못식 공유

공유는 해법이나 손패 순서를 노출하지 않고 결과만 한 줄로 전달한다.

권장 문구:

```text
스택스 #2026071321 ★★ 236점 / 최대 347점 https://게임주소
```

PERFECT 예시:

```text
스택스 #2026071321 PERFECT 347점 https://게임주소
```

동작:

- iOS/Android와 터치 iPad에서는 `navigator.share` 네이티브 공유 시트를 연다.
- 데스크톱에서는 클립보드에 복사하고 `결과를 클립보드에 복사했어요` 토스트를 표시한다.
- 공유 URL은 현재 seed를 강제로 여는 링크가 아니라 기본 게임 주소를 사용한다.
- 공유받은 사용자는 현재 시간 게임을 플레이하고, seed 문자열로 서로 같은 판인지 비교한다.

## 10. 수동 드래그 입력

### 10.1 기본 원칙

- 손패 카드는 자동 배치하지 않는다.
- 손패에서 정원으로 직접 드래그해야 한다.
- 드래그 시작과 동시에 네 정원의 예상 결과를 계산한다.
- 네 정원은 항상 유효 드롭 대상이다.
- 드래그 카드는 보간 애니메이션 없이 같은 프레임의 포인터 위치를 따른다.
- 유효 드롭에서는 놓은 위치에서 대상 슬롯까지 0.18초 이하로 짧게 스냅한 뒤 착지 효과와 효과음을 낸다.

목적지별 미리보기:

- `3장째 · 수확까지 1장`
- `합 19 × 최장 2 = 38`
- `합 19 × 최장 3 = 57`
- `합 19 × 같은 생물군 3 = 57`

자동 추천 테두리는 사용하지 않는다. 점수 결과는 보여주되 선택은 플레이어에게 남긴다.

### 10.2 접근성 대체 입력

`수동`은 시스템이 목적지를 대신 정하지 않는다는 뜻이다. 드래그가 어려운 사용자를 위해 다음 동등 입력은 유지한다.

- 카드 탭 → 정원 탭
- 키보드 `1~5`로 카드 선택 → `Q/W/E/R`로 정원 선택

두 입력 모두 목적지를 플레이어가 직접 확정하므로 자동 배치가 아니다.

## 11. 최신 심플 UI 적용

### 상단

- 게임방법
- 결과공유
- `다음 게임 34:12` 또는 `새 게임 준비됨`

### 목표줄

```text
1별 120  |  2별 200  |  3별 270  |  최대 347
```

현재 점수가 기준을 넘으면 별을 즉시 채운다. 최대 점수는 플레이 전부터 공개한다.

### 보드

- 4정원 `2×2`
- 각 정원은 네 장 슬롯을 고정 크기로 예약
- 별도 순서 안내줄 없이 정원 라벨을 `A B / D C` 위치에 직접 표시
- 정원 연결 후보가 생기면 시작 정원부터 이어지는 꼭대기만 선으로 연결
- 드래그 미리보기는 레이아웃 크기를 바꾸지 않음

### 손패

- 큰 이미지와 숫자 유지
- 곱 계수 제거
- 실제 동식물 이름은 이미지 아래에 크게 표시하고 생물군은 카드 우측 상단에 작게 표시
- 같은 시간 seed에서는 고유 동식물 40종의 종류·숫자 배정과 일러스트를 동일하게 재현
- 드래그 중 원본 슬롯 크기 고정
- `9 정원 카드` 같은 하단 유형 설명 제거
- 게임 시작과 새로받기에서 카드 5장이 순차적으로 들어오는 deal 애니메이션과 효과음 재생
- 유효 드롭의 짧은 스냅이 끝난 뒤 보드 카드와 수확 모션 표시
- 현재 손패를 덱 뒤로 순환시키는 새로받기를 게임당 3회 제공

### 수확 점수 연출

1. 수확된 네 카드를 잠시 보드에 유지하고 놓인 순서대로 카드 위에 `+N`을 300ms 간격으로 표시한다.
2. 카드 연쇄는 수확 카드 목록 중앙에 `×C`, 시계방향 정원 연결은 연결된 각 정원의 꼭대기 카드 위에 누적 배수를 표시하며 전체 경로가 이어지면 `×5`~`×7`까지 계속 증가한다.
3. 같은 생물군 네 장이면 카드 목록 중앙에 `같은 생물군 ×3` 후보를 별도로 표시하고, 수확 연쇄와 비교해 더 높은 배수를 굵게 강조한다.
4. 후보 배수는 서로 곱하지 않는다. 실제 점수에 쓰인 최고 배수만 굵게 강조하고 마지막에 `+획득점수`와 성공 링·스파크를 표시한다.
5. 각 합과 배수 단계의 짧은 효과음은 진행할수록 음높이가 올라가며 마지막 점수에서 확정음을 낸다.
6. 연출 중에는 다음 입력을 잠그고, `prefers-reduced-motion`에서는 중간 단계 없이 최종 점수를 즉시 표시한다.

### 종료

- 최종 점수와 별
- 최대 점수 대비 비율
- 같은 seed 최고 기록
- 게임방법을 통한 다시하기
- 결과공유
- 새 게임이 준비된 경우 새 게임 시작

## 13. P0 검증 항목

규칙 테스트:

- 숫자와 관계없는 네 장 합 수확
- 화면 배치 `A B / D C`와 시계방향 `A → B → C → D` 순서
- 증가·감소·`0↔9` 원형 정원 연결
- 빈 정원, 중복, 비연속, 방향 반전에서 연결 중단
- 놓인 순서의 최장 카드 연쇄와 중복·방향 반전 중단
- seed별 고유 동식물 40종, 생물군 5개마다 서로 다른 여덟 동식물·여덟 숫자, 숫자마다 네 장인 균형 덱
- 지형·관리/도구 종 제외
- 같은 `comboTypeId`의 서로 다른 동식물 네 장 `×3`, 세 장 이하와 3+1 다른 생물군의 미발동
- `네 장 합 × max(카드 연쇄, 정원 연결, 같은 생물군)` 점수와 배수 비중첩
- 연결에 사용된 다른 정원 카드 유지
- 수확 후 해당 정원만 초기화
- 정원당 입력 대기 0~3장 보장
- 40장 종료와 최종 별 판정

시간·저장 테스트:

- KST `YYYYMMDDHH` seed
- 생성기 버전 포함 결정성
- 같은 seed의 카드별 `speciesId`, `comboTypeId`, `variantId`와 일러스트 재현
- 다른 seed의 종류·숫자·일러스트 배정 변형과 중립 이미지 fallback
- 시간 경계에서 현재 런 유지와 pending seed 생성
- 수동 새 게임 전환
- 현재 seed 다시하기와 최고 기록 유지
- 게임당 3회 새로받기의 결정적 순환, 저장·복원
- seed별 저장·복원
- 공유 문구와 모바일/데스크톱 분기

solver 테스트:

- 같은 seed의 동일 최대 점수
- 카드 배치 40수와 새로받기 최대 3회를 포함한 최적 경로 재생 결과와 최대 점수 일치
- 별 컷 10점 단위와 엄격한 순서
- 1,000개 seed의 풀이 시간·탐색 상태 수 상한
- seed 품질 기준 미달 시 salt 재생성

UX 검증:

- 360px에서 카드 드래그와 네 정원 드롭 가능
- 드래그 중 지연 보간 없이 포인터를 따르고 유효 드롭에서만 0.18초 이하 스냅
- 시작·새로받기 deal 모션 중 다음 입력 차단과 종료 후 즉시 복구
- 드래그 시작 후 목적지별 점수 미리보기 100ms 이내 표시
- 자동 추천 없이도 수확 결과 예측 가능
- 새 시간 알림이 현재 게임을 가리지 않음
- 새로고침 후 진행 중 게임 복원

## 14. 성공 기준

정성 기준:

- 플레이어가 `네 장은 합, 시계방향 연결 수는 곱`이라고 설명한다.
- 어느 정원에 드롭할지를 실제 전략으로 느낀다.
- 낮은 점수의 원인을 카드 운보다 꼭대기 숫자 관리에서 찾는다.
- 같은 seed 재도전에서 다른 배치 경로를 시도한다.
- 시간이 지나도 현재 게임이 사라지지 않는다는 것을 이해한다.

초기 지표:

- 첫 판 완료율: 90% 이상
- 1별 도달률: 80~95%
- 2별 도달률: 45~70%
- 3별 도달률: 10~25%
- 같은 seed 재도전율: 40% 이상
- 결과 공유율: 5% 이상
- 시간 경계 새 게임 전환율: 30% 이상
- 드래그 취소·오드롭률: 5% 이하

## 15. 최종 판단

이 제안은 `수확 솔리테어`보다 정확히는 **매시간 제공되는 4정원 점수 퍼즐**에 가깝다. 그러나 Stacks의 핵심 자산인 숫자 카드, 네 정원, 손패, 수확 애니메이션을 더 짧고 강한 규칙으로 묶는다.

가장 좋은 점은 선택과 결과가 같은 화면에 있다는 것이다. 플레이어는 카드 한 장을 직접 드래그하면서 한 정원 안의 카드 순서, 시계방향 정원 연결, 같은 생물군 네 장 수집 사이를 저울질한다. 네 번째 카드가 즉시 수확되므로 놓을 수 없는 패는 없고, 낮은 점수만 있어 고전 솔리테어의 불공정한 막힘도 사라진다.

시간 seed의 정확한 최대 점수와 별 목표, 말잇못식 수동 새 게임 전환, 같은 seed 재도전, 한 줄 공유를 결합하면 3~5분짜리 반복 도전 구조가 완성된다.

**권장 최종 규칙은 `네 장 = 합`, `수확 연쇄 = ×1~×7`, `같은 생물군 네 장 = ×3`, `후보 중 최고 하나만 적용`, `40장 종료 = 별 평가`, `모든 생물군·종 배정과 카드 순서 = 시간 seed로 재현`이다.**
