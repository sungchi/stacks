# Stacks (스택스) UI 제안 보고서

> 작성일: 2026-07-04  
> 범위: Stacks (스택스) 웹 재구현의 플레이 화면, 보조 모달, 모바일 우선 UI 방향
> 근거: [AGENTS.md](../../AGENTS.md), [docs/requirements.md](../requirements.md), [src/app.js](../../src/app.js), [src/styles.css](../../src/styles.css), [docs/screenshots/](../screenshots/)

## 1. 결론

Stacks (스택스)의 UI는 “카드를 골라 4개 정원 더미 중 어디에 놓을지 판단하는 게임”으로 즉시 읽혀야 한다. 가장 깔끔하고 직관적인 방향은 대시보드형 정보 UI가 아니라, **손패와 4정원 더미가 중심이 되는 조용한 전술 테이블 UI**다.

현재 구현은 핵심 요소가 모두 들어가 있고 데스크톱에서는 게임 구조가 잘 드러난다. 다만 모바일에서는 정보와 메뉴가 앞쪽에 많이 쌓이며, 추천/상태/보조 시스템이 플레이 테이블보다 먼저 읽히는 구간이 있다. 다음 단계에서는 “지금 할 수 있는 한 수”를 중심으로 시각 위계를 다시 잡는 것이 좋다.

## 2. 관찰한 사실

- 첫 화면은 실제 게임 화면으로 시작하며, 상단 메뉴, 좌측 런 상태, 추천 패널, 4개 더미, 손패가 한 화면에 배치되어 있다.
- `src/app.js` 기준 플레이 화면은 `topbar -> left-rail -> table-surface -> piles -> hand` 구조다.
- 손패 카드는 숫자, 이미지, 이름, 유형/색상, 예상 평판을 동시에 보여준다.
- 더미는 흐름 수, 최근 카드 스택, 현재 맨 위 카드, 예상 평판, 배율을 함께 보여준다.
- 추천 패널은 항상 별도 패널로 노출되며, 자동 놓기 버튼을 제공한다.
- 상점과 도감은 보드 위 모달로 열려 현재 보드 맥락을 유지한다.
- 드래그, 착지, 점수 팝, 무효 흔들림, 모션 끄기 설정은 이미 UI 계층에 구현되어 있다.
- 저장된 모바일 스크린샷에서는 상단 메뉴와 추천 패널의 가로 폭이 좁은 화면에서 답답하게 읽힌다. 일부 정보는 플레이 테이블보다 먼저 큰 면적을 차지한다.

## 3. UI 목표

좋은 Stacks (스택스) UI는 플레이어에게 매 순간 하나의 질문만 또렷하게 던져야 한다.

**“이 카드가 어느 정원에 가장 잘 이어질까?”**

이 질문을 위해 필요한 정보는 먼저 보여주고, 나머지는 행동 직전이나 행동 후에만 펼치는 것이 좋다.

핵심 원칙:

- 카드와 더미가 화면의 주인공이어야 한다.
- 추천은 정답처럼 보이지 않고 부드러운 힌트처럼 보여야 한다.
- 숫자 계산은 “결과 먼저, 이유는 나중” 순서로 읽혀야 한다.
- 모바일에서는 상단 설명보다 손패, 더미, 현재 목표가 먼저 닿아야 한다.
- 보조 시스템인 상점, 도감, 스테이지, 설정은 플레이를 방해하지 않는 위치와 밀도로 둔다.

## 4. 제안 레이아웃

### 4.1 플레이 화면

권장 구조:

1. 상단 상태 스트립
2. 4정원 더미 영역
3. 선택 카드 상세 또는 추천 한 줄
4. 하단 손패 영역

상단 상태 스트립에는 다음만 남긴다.

- 현재 스테이지/지형
- 평판 진행도
- 남은 내기
- 남은 갈아엎기

`런 코인`, `최고 흐름`, `씨앗`은 항상 필요한 정보가 아니다. 기본 화면에서는 접거나 작게 두고, 상점/결과/보상 단계에서 더 크게 보여주는 편이 낫다.

### 4.2 손패

손패 카드는 “선택 가능한 물체”로 먼저 읽혀야 한다. 현재처럼 카드 안에 모든 속성을 넣으면 전략 정보는 풍부하지만, 모바일에서는 카드가 설명 카드처럼 보일 위험이 있다.

권장 카드 표면:

- 크게: 숫자, 픽셀 아트, 카드명
- 작게: 예상 평판
- 숨김 또는 축약: 유형, 색상

유형/색상은 텍스트 대신 작은 아이콘이나 색 테두리로 보내고, 선택 후 상세 패널에서 이름을 풀어쓰는 것이 좋다.

### 4.3 더미

더미는 “놓을 수 있는 자리”로 즉시 보여야 한다. 기본 상태에서는 정원 번호, 흐름 수, 최근 카드만 보여주고, 카드가 선택되거나 드래그될 때만 예상 결과를 강조한다.

선택/드래그 중 더미 표시:

- 추천 더미: 얇은 금색 테두리와 `+점수`
- 흐름이 끊기는 더미: 낮은 채도의 경고 테두리
- 드롭 중: 더미 위 작은 배지로 `+점수`, `흐름`, `x배율`

이렇게 하면 네 더미 모두가 계속 계산표처럼 보이지 않고, 플레이어가 움직일 때 필요한 정보만 살아난다.

### 4.4 추천

추천은 독립 패널보다 “선택 카드의 보조 정보”로 붙는 편이 직관적이다.

권장 형태:

- 손패 위 또는 더미 아래 한 줄: `추천: 나비 -> 정원 1, +21`
- 버튼명: `추천 놓기`
- 강조도: 기본 버튼보다 약하게

자동 플레이 버튼은 편리하지만 게임을 대신 풀어주는 느낌이 강해질 수 있다. 버튼은 남기되, 시각적으로는 카드와 더미 선택보다 한 단계 낮춰야 한다.

### 4.5 보조 메뉴

상단의 `스테이지`, `상점`, `도감`, `설정`은 데스크톱에서는 괜찮지만, 모바일에서는 첫 화면의 시선을 많이 가져간다.

권장 형태:

- 데스크톱: 현재처럼 우측 상단 메뉴 유지
- 모바일: `스테이지`, `상점`, `도감`은 작은 탭 또는 메뉴 버튼 안으로 축약
- 설정은 아이콘 버튼 유지

플레이 화면에서 가장 큰 버튼은 “다음 행동”이어야 한다. 보조 메뉴 버튼이 카드/더미보다 먼저 커 보이지 않게 조정한다.

## 5. 화면별 제안

### 5.1 모바일 플레이

우선순위가 가장 높다. 모바일에서는 한 화면에 모든 것을 넣기보다, 위에서 아래로 “상태 -> 더미 -> 손패”가 자연스럽게 이어져야 한다.

권장 변경:

- 상단 브랜드 영역 높이를 줄인다.
- 메뉴 버튼은 한 줄 작은 탭 또는 단일 메뉴로 축약한다.
- 런 상태 카드는 세로 큰 패널이 아니라 낮은 상태 스트립으로 바꾼다.
- 추천 패널은 손패 근처의 한 줄 힌트로 이동한다.
- 더미 4개는 2x2로 유지하되, 카드가 선택되기 전에는 점수/배율 정보를 덜 보인다.
- 손패는 하단에 고정 또는 준고정 배치해 매 턴 손이 가는 위치를 일정하게 만든다.

검증 기준:

- 360px 폭에서 가로 스크롤이 없어야 한다.
- 첫 화면에서 정원 더미 하나 이상과 손패 일부가 동시에 보여야 한다.
- 추천 버튼 텍스트가 잘리거나 화면 밖으로 밀리지 않아야 한다.

### 5.2 데스크톱 플레이

데스크톱은 현재 구조가 비교적 안정적이다. 다만 좌측 레일의 정보가 조금 강해서, 테이블을 더 넓고 차분하게 보이도록 조정하면 좋다.

권장 변경:

- 좌측 레일 폭을 줄이고, 핵심 상태만 남긴다.
- 추천 패널을 테이블 하단 또는 손패 툴바로 옮긴다.
- 더미 카드의 빈 공간은 유지하되, 프리뷰 정보는 선택/드래그 상태에서만 강하게 보인다.
- 손패 카드의 유형/색상 텍스트를 작은 칩이나 아이콘으로 줄인다.

### 5.3 상점

상점은 “이번 런 구매”와 “메타 성장”을 나눈 점이 좋다. 다만 구매 전후 변화가 더 즉시 읽혀야 한다.

권장 변경:

- 구매 가능한 항목을 위쪽에 더 강하게 배치한다.
- 각 항목의 설명은 한 줄 효과와 다음 값만 남긴다.
- 구매 후에는 해당 행에서 바로 변화량을 짧게 강조한다.
- 잠긴 메타 항목은 너무 흐리게 만들기보다, 필요한 재화와 기대 효과는 읽히게 둔다.

### 5.4 도감

도감은 “다음 발견”을 크게 보여주는 방향이 좋다. 이 게임의 장기 동기를 설명하는 데 효과적이다.

권장 변경:

- 상단의 다음 발견 카드를 유지한다.
- 진행 탭은 숫자보다 시각적 진행 막대나 작은 스탬프 느낌을 강화한다.
- 숨김 항목은 너무 많은 `???`보다 카테고리별 남은 수를 먼저 보여준다.
- 카드 숫자 필터는 모바일에서 가로 스크롤 칩으로 제공한다.

### 5.5 보상과 결과

보상/결과 화면은 현재 보드 위 모달로 뜨는 방식이 좋다. 플레이 맥락을 잃지 않기 때문이다.

권장 변경:

- 보상 선택지는 “지금 효과”와 “앞으로 좋아지는 점”을 분리한다.
- 결과 화면은 성공/실패 판단, 다음 버튼, 상점 버튼만 먼저 보여준다.
- 세부 기록은 접힌 영역으로 둔다.

## 6. 구현 우선순위

### P0. 모바일 가로 넘침과 첫 화면 위계 정리

- 모바일에서 상단 메뉴와 추천 패널이 화면 폭을 넘지 않게 한다.
- HUD를 낮은 상태 스트립으로 바꾼다.
- 추천을 손패 근처 한 줄 힌트로 옮긴다.

완료 기준:

- 360px, 390px 화면에서 가로 스크롤 없음.
- 플레이 시작 직후 카드/더미/목표가 모두 한 번에 이해된다.

### P1. 카드와 더미 정보 밀도 낮추기

- 카드 표면에서 유형/색상 텍스트를 축약한다.
- 더미 프리뷰는 선택/드래그 중에만 강하게 표시한다.
- 선택 카드 상세 패널을 추가해 계산 이유를 한곳에 모은다.

완료 기준:

- 카드 하나를 봤을 때 숫자와 이미지가 먼저 읽힌다.
- 더미 하나를 봤을 때 “현재 흐름”과 “놓으면 얻는 점수”가 구분된다.

### P2. 추천과 드래그 피드백 정리

- 추천 놓기 버튼의 강조도를 낮춘다.
- 드래그 중 더미별 결과 배지를 더 명확하게 한다.
- 무효 드롭 시 왜 무효인지 짧은 힌트를 준다.

완료 기준:

- 드래그 중 놓을 더미가 즉시 구분된다.
- 추천은 보이지만 수동 선택을 방해하지 않는다.

### P3. 상점/도감/결과 polish

- 상점 구매 후 변화량을 즉시 강조한다.
- 도감의 다음 발견 목표를 더 선명하게 한다.
- 결과 화면은 다음 행동 중심으로 줄인다.

완료 기준:

- 보조 화면을 닫고 돌아왔을 때 현재 런 상황을 잃지 않는다.
- 각 모달에서 가장 중요한 행동 버튼이 하나로 읽힌다.

## 7. 권장 디자인 톤

시각 톤은 “따뜻한 픽셀 정원 테이블”이 가장 어울린다. 밝은 종이색, 깊은 녹색 보드, 작은 금색 강조는 유지하되, 모든 요소에 동시에 강조색을 주지 않는다.

권장 톤:

- 배경: 밝고 조용한 정원 노트 느낌
- 보드: 깊은 녹색 테이블
- 카드: 흰색 종이 카드와 선명한 픽셀 아트
- 강조: 현재 선택, 추천, 드롭 가능 위치에만 금색 사용
- 경고: 흐름 끊김과 무효 드롭에만 낮은 채도의 붉은색 사용

피해야 할 것:

- 모든 카드와 더미에 점수/배율/속성 텍스트를 계속 강하게 노출하는 방식
- 상단 메뉴가 플레이 액션보다 커지는 방식
- 모바일에서 정보 패널이 보드를 화면 아래로 밀어내는 방식
- 추천 버튼이 사실상 “정답 버튼”처럼 보이는 방식

## 8. 최종 판단

Stacks (스택스)의 장점은 규칙의 복잡함보다 “작은 카드 한 장을 어디에 얹을까”라는 손맛에 있다. UI도 그 손맛을 따라가야 한다.

가장 좋은 다음 UI 개편은 새 기능 추가가 아니라 정보의 순서를 정리하는 일이다. 목표와 남은 행동은 작게, 손패와 더미는 크게, 추천과 계산식은 상황에 따라 나타나게 만들면 게임이 훨씬 직관적으로 느껴질 것이다.
