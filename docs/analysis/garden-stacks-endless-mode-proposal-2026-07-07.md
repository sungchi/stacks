# Garden Stacks 무한 정원 모드 제안서

> 작성일: 2026-07-07  
> 개정: 2026-07-07, 계절 전환과 지형 피로도를 제외하고 수확 일괄 점수 정산으로 단순화  
> 개정 2: 2026-07-07, 더미별 덧셈 수와 곱셈 계수의 콤보 누적/초기화 규칙 추가  
> 범위: Garden Stacks에 수박게임식 고득점 추구 모드를 추가할 때의 규칙, 제한, 재미요소 제안  
> 근거: [AGENTS.md](../../AGENTS.md), [docs/requirements.md](../requirements.md), [src/game/number-solitaire.js](../../src/game/number-solitaire.js), [src/game/catalog.js](../../src/game/catalog.js)

## 1. 결론

무한모드는 스테이지 목표와 내기 제한을 제거한 캠페인 변형이 아니라, **수확할 때만 점수가 확정되는 고득점 정리 모드**가 되는 편이 좋다.

추천 방향은 이전안보다 더 단순하다.

- 계절 전환은 넣지 않는다.
- 지형 피로도도 넣지 않는다.
- 배치할 때는 점수를 바로 얻지 않는다.
- 카드 배치는 더미 안에 `덧셈 수`와 `곱셈 계수`를 쌓는 행동이다.
- 콤보가 이어지면 두 값이 계속 누적된다.
- 콤보가 깨지면 두 값은 현재 올린 카드의 값으로 초기화된다.
- 일정한 콤보 패턴이 완성되면 수확이 발생한다.
- 수확 순간에만 점수를 한 번에 정산한다.
- 수확된 원본 카드는 사라지지 않고 퇴비 더미로 갔다가 나중에 다시 덱으로 돌아온다.
- 실패 압력은 더미 용량, 손패 막힘, 제한된 갈아엎기에서 온다.

이 구조가 수박게임에 더 가깝다. 수박게임은 매 순간 복잡한 점수 계산을 보여주지 않고, 공간을 비우는 순간의 보상과 남은 공간 압박으로 재미를 만든다. Garden Stacks도 매 배치 점수보다 “언제 수확해서 공간과 점수를 얻을까”가 중심이 되는 편이 더 선명하다.

## 2. 현재 게임 기준

현재 기준 게임은 스테이지형 숫자 솔리테어다.

- 플레이어는 0~9 숫자 카드를 4개 정원 더미에 놓는다.
- 같은수, 이웃수, 합 9, 홀짝, 더블 관계로 평판을 얻는다.
- 콤보 유지 조건은 이웃수다.
- 초지, 숲, 습지는 각각 이웃수, 같은수/홀짝, 합 9를 선호한다.
- 기본 스테이지는 목표 평판, 내기 수, 갈아엎기 수가 정해져 있다.
- 첫 5콤보와 스테이지 클리어는 발견 보상으로 이어진다.

무한모드는 이 기반에서 `숫자 관계를 읽는 맛`과 `4개 더미 중 어디에 쌓을지 판단하는 맛`만 가져온다. 목표 평판, 내기 제한, 지형별 장기 규칙은 빼고, 더미 용량과 수확 정산을 중심으로 다시 잡는다.

## 3. 모드 정체성

모드명은 **무한 정원**을 권장한다.

한 줄 피치:

**4개의 정원 더미가 가득 차기 전에 숫자 흐름을 수확으로 비우고, 수확 순간에 점수를 한 번에 얻으며 최고 점수를 노리는 고득점 카드 모드.**

이 모드는 캠페인을 대체하지 않는다. 캠페인은 목표 달성형, 무한 정원은 생존형 점수 모드로 분리한다.

| 항목 | 캠페인 | 무한 정원 |
|---|---|---|
| 목표 | 제한 내기 안에 목표 평판 달성 | 가능한 오래 버티며 최고 점수 갱신 |
| 점수 | 배치마다 평판 획득 | 수확 순간에만 점수 획득 |
| 실패 압력 | 내기 수, 목표 미달 | 더미 용량, 손패 막힘, 갈아엎기 소진 |
| 보상 타이밍 | 첫 5콤보, 스테이지 클리어 | 수확, 최고 수확, 점수 구간 |
| 재미 | 퍼즐 최적화와 성장 | 공간 압박, 정리 쾌감, 한 수 더 욕심 |
| 저장 | 캠페인 진행과 현재 런 | 최고 점수, 일일 씨앗 기록, 현재 런 |

## 4. 기본 규칙 제안

### 4.1 시작 상태

MVP 기본값:

- 모드: `endless`
- 더미 수: 4
- 시작 덱: 0~9 카드 2벌, 총 20장
- 손패: 5장
- 시작 갈아엎기: 3
- 목표 평판: 없음
- 내기 제한: 없음
- 정원 더미 용량: 더미마다 일반 카드 9칸
- 다음 카드 예고: 덱 위 3장 표시
- 점수: 0

기존 캠페인의 카드 표면 언어는 버리지 않는다. 단, 무한모드에서는 배치 즉시 점수를 올리지 않고, 카드의 `덧셈 수`와 `곱셈 계수`를 더미의 현재 흐름 값에 반영한다.

### 4.2 배치와 더미 점수 값

카드를 더미에 놓을 때는 총점이 오르지 않는다.

각 카드는 두 개의 점수 값을 가진다.

- `덧셈 수`: 카드가 현재 흐름에 더하는 값. 카드 좌상단의 합 숫자와 같은 계열이다.
- `곱셈 계수`: 카드가 현재 흐름의 곱셈값에 더하는 값. 카드 우하단의 곱 숫자와 같은 계열이다.
- MVP 기본값은 `덧셈 수 = 카드 숫자`, `곱셈 계수 = 9 - 카드 숫자`다.
- 보너스, 반짝 카드, 보상 효과는 우선 덧셈 수에 더하고, 곱셈 계수 보정은 별도 보상으로 제한한다.

각 더미는 현재 흐름에 대해 다음 값을 들고 있다.

- `누적 덧셈 수`
- `누적 곱셈 계수`
- `현재 콤보 길이`

배치 처리:

- 손패 카드 1장을 더미에 놓는다.
- 이전 맨 위 카드와의 숫자 관계를 평가한다.
- 콤보가 이어지면 `누적 덧셈 수 += 현재 카드 덧셈 수`, `누적 곱셈 계수 += 현재 카드 곱셈 계수`로 처리한다.
- 콤보가 깨지면 `누적 덧셈 수 = 현재 카드 덧셈 수`, `누적 곱셈 계수 = 현재 카드 곱셈 계수`로 초기화한다.
- 콤보가 깨졌을 때 이전 흐름의 미수확 점수 값은 버려지고, 이전 카드들은 더미 공간만 차지한다.
- 더미의 수확 패턴 상태를 갱신한다.
- 손패를 보충한다.
- 현재 점수는 변하지 않는다.

MVP에서 콤보 유지 조건은 캠페인과 같이 `이웃수`다. 이후 수확 패턴이 늘어나더라도, 기본 흐름 유지 조건은 하나로 유지하는 편이 읽기 쉽다.

UI는 배치 전후에 다음을 보여준다.

- `덧셈 +7`
- `곱셈 +2`
- `수확 예상 63`
- `수확까지 2장`
- `수확 가능`

핵심은 플레이어가 매 턴 작은 점수를 받는 느낌이 아니라, 더미 안의 덧셈 수와 곱셈 계수가 점점 커지고 있다고 느끼게 하는 것이다. 콤보가 깨지면 이 값이 현재 카드 기준으로 작아지므로, “한 장 더 안전하게 이어갈까, 다른 더미에 새 흐름을 만들까”가 매 턴의 판단이 된다.

### 4.3 덱 순환과 카드 재사용

수확된 카드는 다시 사용할 수 있어야 한다. 이 규칙은 무한모드와 잘 맞는다.

- 시작 덱은 0~9 카드 2벌이다.
- 덱 위 3장은 예고로 보여준다.
- 손패를 갈아엎은 카드는 퇴비 더미로 간다.
- 수확으로 제거된 원본 카드도 퇴비 더미로 간다.
- 현재 덱이 비면 퇴비 더미를 섞어 새 덱으로 만든다.
- 덱과 퇴비 더미가 모두 비는 예외 상황이면 기본 0~9 두 벌을 다시 만든다.

이렇게 하면 수확은 공간을 비우는 행동이면서 동시에 좋은 숫자들을 나중에 다시 순환시키는 행동이 된다. 다만 수확 직후 같은 카드가 바로 손패로 돌아오면 반복 루프가 쉬워지므로, 수확 카드는 즉시 덱 위에 올리지 않고 퇴비 더미를 거쳐야 한다.

### 4.4 더미 용량

각 정원 더미는 일반 카드 기준 9칸을 가진다.

- 비어 있는 더미에는 어떤 카드든 놓을 수 있다.
- 용량 미만인 더미에는 카드를 놓을 수 있다.
- 용량에 도달한 더미는 `가득 참` 상태가 된다.
- 가득 찬 더미에는 이번 배치로 즉시 수확을 만들 수 있는 카드만 놓을 수 있다.
- 가득 찬 더미에 수확을 만들 수 없으면 그 더미는 현재 손패 기준으로 막힌 자리다.

용량은 무한모드의 핵심 압박이다. 수박게임에서 공간이 차오르듯, Garden Stacks에서는 더미가 차오른다. 점수는 수확 때만 들어오므로, 수확하지 못한 고점 더미는 동시에 위험한 더미가 된다.

### 4.5 수확 조건

MVP에서는 수확 조건을 적게 시작한다. 첫 버전은 `이웃수 5장`만으로 충분하다.

MVP 수확:

- 한 더미의 최근 연결 카드가 이웃수 흐름으로 5장 이어지면 자동 수확한다.
- 수확 대상은 최근 연결 카드 5장이다.
- 수확된 원본 카드 5장은 더미에서 제거되어 퇴비 더미로 이동한다.
- 수확 후에는 제거된 5장 아래에 있던 카드가 새 맨 위 카드가 된다.
- 아래 카드가 없으면 해당 더미는 빈 더미가 된다.
- 수확은 새 카드를 만들지 않는다.

MVP 이후 추가 수확 패턴:

| 수확 | 조건 | 의도 |
|---|---|---|
| 길 수확 | 이웃수 5장 | 기본 압축, 가장 직관적 |
| 반사 수확 | 최근 4장 안에 합 9가 2회 | 합 9 루트의 고점 |
| 나이테 수확 | 같은수 3장 | 반복 루트의 빠른 압축 |
| 사계 수확 | 한 더미에서 4색 완성 | 색상 접목 보상의 목적 강화 |
| 생태 수확 | 지형 -> 식물 -> 동물 -> 관리/도구 순서 완성 | 유형 순서 보상의 목적 강화 |

처음부터 모든 수확 패턴을 넣으면 UI가 무거워진다. P0는 길 수확 하나로 시작하고, 나머지는 보상이나 확장 규칙으로 여는 편이 낫다.

### 4.6 수확 점수

점수는 수확 시 한 번에 들어온다.

기본 공식:

```text
수확 점수 = 누적 덧셈 수 × 누적 곱셈 계수
```

예:

```text
현재 흐름 누적 덧셈 수: 24
현재 흐름 누적 곱셈 계수: 7
수확 점수: 24 × 7 = 168
```

정산 원칙:

- 배치 시점에는 점수 증가 없음.
- 수확 시점에만 총점 증가.
- 수확되지 않은 카드는 점수로 확정되지 않는다.
- 콤보가 깨지면 이전 흐름의 누적 덧셈 수와 누적 곱셈 계수는 버려진다.
- 게임 오버 시 더미에 남은 미수확 점수 값은 기본적으로 버린다.
- 보상 중 하나로 `마지막 정리`를 추가해 남은 미수확 점수 일부를 얻는 변형은 가능하다.

이 방식은 “터뜨리기 전까지는 내 점수가 아니다”라는 긴장을 만든다. 높은 덧셈 수와 곱셈 계수를 쌓은 더미일수록 수확 전에는 달콤하면서도 위험하다.

## 5. 실패 조건

권장 실패 조건:

- 손패의 어떤 카드도 4개 더미 중 합법적으로 놓을 수 없다.
- 갈아엎기 또는 긴급 도구가 남아 있지 않다.
- 갈아엎어도 합법 배치를 만들 수 있는 새 손패가 없다.

합법 배치 판정은 다음을 본다.

- 더미가 비어 있는가.
- 더미가 용량 미만인가.
- 더미가 가득 찼다면 이번 배치로 수확이 발생하는가.

`게임 오버` 화면에는 최종 점수와 함께 실패 이유를 보여준다.

예:

- `정원 2와 4가 가득 찼고, 남은 손패가 수확을 만들 수 없습니다.`
- `갈아엎기를 모두 사용했고 놓을 수 있는 카드가 없습니다.`
- `높은 덧셈 수와 곱셈 계수를 쌓았지만 수확하지 못했습니다.`

## 6. 점수와 기록

표시 우선순위:

1. 현재 점수
2. 최고 점수
3. 선택 카드의 덧셈 수와 곱셈 계수
4. 선택 더미의 수확 예상 점수
5. 이번 판 최고 수확 점수
6. 생존 턴

로컬 기록은 최소한 다음을 저장한다.

- 최고 점수
- 최고 생존 턴
- 최고 수확 점수
- 최고 미수확 예상 점수
- 마지막 플레이한 무한 정원 런
- 일일 씨앗 최고 점수

경쟁성을 넣는다면 `일반 기록`과 `성장 적용 기록`을 분리해야 한다. 캠페인 메타 업그레이드가 무한모드 점수에 직접 영향을 주면 최고 점수 비교가 흐려진다.

## 7. 갈아엎기와 상점

무한모드의 갈아엎기는 탈출 버튼이다. 강하지만 제한적이어야 한다.

권장 규칙:

- 시작 갈아엎기 3회.
- 수확 3회마다 갈아엎기 +1, 최대 보유 5회.
- 갈아엎기는 손패 전체 교체를 기본으로 한다.
- 가득 찬 더미 때문에 막힌 손패가 있을 때 갈아엎기 버튼을 더 강하게 표시한다.
- 갈아엎은 카드는 퇴비 더미로 간다.

런 상점은 작게 시작한다.

MVP 상점 후보:

- 과밀 정원 오래된 카드 제거.
- 다음 카드 1장 구매.
- 손패 1장 반짝 강화.
- 덱에서 가장 많은 숫자 1장 제거.
- 다음 수확 점수 +20%.
- 퇴비 더미를 즉시 섞어 덱 아래에 붙이기.

상점 코인은 수확으로만 지급하는 편이 좋다.

- 수확 1회당 코인 +1.
- 큰 수확 점수 달성 시 추가 코인 +1.
- 실패 직전 회복 보상은 코인을 직접 주지 않는다.

## 8. 보상과 발견

무한모드 보상은 캠페인의 발견 보상을 재사용하되, 발동 타이밍과 효과를 수확 중심으로 바꾼다.

보상 발동:

- 첫 수확.
- 새 최고 수확 점수.
- 큰 수확 점수 달성.
- 점수 구간 돌파.
- 처음 만든 수확 패턴.

보상 종류:

- 카드 추가: 특정 숫자를 덱 또는 퇴비 더미에 추가.
- 덱 정리: 가장 많은 숫자 제거.
- 콤보 강화: 특정 수확 패턴의 덧셈 수 또는 곱셈 계수 보정.
- 수확 강화: 다음 수확 점수 증가 또는 수확 조건 완화.
- 용량 보정: 한 더미의 이번 런 용량 +1, 최대 +2.
- 예고 강화: 다음 5턴 동안 덱 예고 +1.
- 퇴비 순환: 퇴비 더미 일부를 덱 아래로 즉시 이동.

도감은 “무한 정원 기록장”으로 확장할 수 있다.

- 최고 수확 점수 스탬프.
- 수확 패턴별 첫 발견.
- 숫자별 마지막으로 수확된 카드.
- 최고 미수확 예상 점수 기록.
- 일일 씨앗 기록.

## 9. 재미요소

### 9.1 한 수 더 욕심

수확 점수를 한 번에 정산하면, 플레이어는 매번 “수확 가능한 더미를 안전하게 비울까, 다른 더미에 더 큰 덧셈 수와 곱셈 계수를 쌓을까”를 고민하게 된다.

추천 방향:

- 5장 수확은 자동.
- 수동 수확은 MVP에 넣지 않는다.
- “더 키우기” 선택도 MVP에 넣지 않는다.
- 고점 욕심은 수동 버튼이 아니라 어느 더미에 덧셈 수와 곱셈 계수를 쌓을지에서 나오게 한다.

### 9.2 위기 탈출

가득 찬 더미에서 딱 맞는 숫자를 뽑아 수확을 만드는 순간이 핵심 재미가 된다.

필요한 피드백:

- 가득 찬 더미는 조용하지만 분명한 위험 테두리.
- 손패 카드 선택 시 `수확 가능` 더미는 가장 먼저 강조.
- 수확 발생 시 최근 카드들이 하나로 접히는 모션.
- 점수는 수확 순간에 큰 숫자 하나로 표시.
- 이유는 `길 수확`, `24 x 7`, `5장 정리` 같은 작은 배지로 표시.

### 9.3 일일 정원

고득점 모드는 비교할 대상이 있을 때 오래 간다. 서버 랭킹이 없어도 일일 씨앗은 가치가 있다.

일일 정원 규칙:

- 날짜 기반 고정 seed를 사용한다.
- 모든 플레이어가 같은 시작 덱 순서를 받는다.
- 메타 업그레이드는 적용하지 않는다.
- 하루 최고 점수, 최고 수확, 생존 턴을 로컬 저장한다.

공유 문구 예:

```text
무한 정원 2026-07-07
점수 12,840 · 최고 수확 840 · 72턴 생존
```

## 10. 제한과 밸런스 가드레일

무한모드는 작은 허점이 무한 루프가 되기 쉽다. 다음 제한을 처음부터 둔다.

- 손패는 기본 5장, 보상 포함 최대 6장.
- 갈아엎기는 최대 5회 보유.
- 더미 용량 증가는 더미당 최대 +2.
- 덱 정리는 최소 덱 크기 14장 아래로 내려가지 못한다.
- 같은 보상은 짧은 구간 안에 반복 출현하지 않는다.
- 같은 더미에서 같은 수확 패턴만 반복하면 보상 후보가 점차 약해진다.
- 수확된 원본 카드는 즉시 손패나 덱 위로 가지 않는다.
- 캠페인 메타 업그레이드는 기본 랭킹에 적용하지 않는다.
- 저장된 무한 런은 하나만 유지하고, 새 일일 정원을 시작하면 별도 확인을 요구한다.

특히 조심할 것은 덱 압축, 퇴비 순환, 갈아엎기 보충이다. 셋이 함께 강하면 원하는 숫자를 너무 쉽게 반복해서 수확할 수 있다.

## 11. UI 요구

무한 정원 첫 화면도 설명 페이지가 아니라 실제 게임 화면이어야 한다.

필수 표시:

- 현재 점수와 최고 점수.
- 생존 턴.
- 4개 정원 더미.
- 더미별 용량 상태.
- 더미별 수확 진행도.
- 더미별 누적 덧셈 수와 곱셈 계수.
- 손패 5장.
- 다음 카드 예고 2~3장.
- 갈아엎기 수.
- 수확 가능 표시.

표시 원칙:

- 현재 점수는 상단에 크게 둔다.
- 배치 카드의 덧셈 수와 곱셈 계수는 작게 둔다.
- 수확 예상 점수는 선택 중인 더미에만 강하게 보여준다.
- 더미 위험은 색보다 선, 배지, 작은 게이지로 읽힌다.
- 손패 카드 선택 시 각 더미의 결과를 한 줄로만 보여준다.
- 수확 가능 더미는 추천보다 더 강하게 표시한다.
- 수확 순간에만 큰 점수 팝을 보여준다.
- 게임 오버 직전에는 `막힌 이유`가 카드와 더미 근처에 떠야 한다.

모바일에서는 점수, 더미, 손패가 한 화면에서 이어져야 한다. 상점, 도감, 설정, 기록은 메뉴 안으로 접어도 된다.

## 12. 구현 제안

무한모드는 기존 규칙 모듈을 갈아엎지 말고, `number-solitaire`의 평가 함수를 최대한 재사용하는 방식이 좋다.

추가할 순수 규칙 후보:

- `newEndlessRun(opts)`
- `evaluateEndlessPilePlay(state, pileIndex, card)`
- `playEndlessCardToPile(state, handIndex, pileIndex)`
- `checkEndlessEnd(state)`
- `detectEndlessHarvest(state, pileIndex)`
- `applyEndlessHarvest(state, pileIndex, pattern)`
- `refillEndlessDeck(state)`
- `endlessRewardOptions(state, reason)`
- `recordEndlessResult(profile, state)`

상태 필드 후보:

```js
{
  mode: "endless",
  score,
  bestScore,
  turnCount,
  pileCapacity,
  compostPile,
  forecast,
  piles: [
    {
      cards,
      addTotal,
      multiplyTotal,
      comboStep,
      harvestHistory
    }
  ],
  endlessRewardsSeen,
  dailySeed
}
```

자동 테스트 필수 범위:

- 시작 상태 생성.
- 배치 시 총점이 변하지 않음.
- 콤보가 이어지면 덧셈 수와 곱셈 계수가 누적됨.
- 콤보가 깨지면 덧셈 수와 곱셈 계수가 현재 카드 값으로 초기화됨.
- 더미 용량 판정.
- 가득 찬 더미의 수확 가능 배치 제한.
- 5장 수확 제거.
- 수확 시에만 점수 증가.
- 수확 점수가 `누적 덧셈 수 × 누적 곱셈 계수`로 계산됨.
- 수확된 원본 카드가 퇴비 더미로 이동.
- 덱이 비면 퇴비 더미가 섞여 재사용됨.
- 수확 후 더미 맨 위 카드 갱신.
- 갈아엎기 최대치와 보충.
- 게임 오버 판정.
- 최고 점수 저장.
- 일일 seed 결정성.

- 콤보 끊김 시 현재 카드 값으로 초기화.
- 더미 용량 9.

## 14. 최종 판단

개정 방향이 더 좋다. 계절 전환, 지형 피로도, 꽃피움 카드는 게임을 풍성하게 만들 수 있지만, 무한모드의 첫 재미에는 필요하지 않다. 이미 숫자 관계, 4더미 용량, 수확 제거, 카드 재순환, 갈아엎기만으로 수박게임 수준의 의사결정 밀도는 나온다.

특히 **덧셈 수와 곱셈 계수를 쌓고, 수확 시 한 번에 정산하는 규칙**은 이 모드의 정체성을 선명하게 만든다. 배치는 점수를 버는 행동이 아니라 점수를 익히는 행동이고, 수확은 그 위험을 확정 보상으로 바꾸는 행동이다. 이 한 문장이 잡히면 무한 정원은 캠페인과 다른 모드로 충분히 설 수 있다.
