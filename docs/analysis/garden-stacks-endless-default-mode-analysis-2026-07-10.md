# Stacks (스택스) 무한 정원 기본 모드 분석

> 작성일: 2026-07-10
> 범위: 현재 `endless` 규칙, 자동 배치, 수확 점수, 특수 능력, 심플 UI, 저장·진입 구조
> 근거: [요구사항](../requirements.md), [게임 규칙 구현](../../src/game/number-solitaire.js), [심플 UI](../../src/simple-app.js), [자동 테스트](../../tests/number-solitaire.test.mjs)

## 1. 한 줄 결론

**무한 정원은 Stacks의 기본 모드로 삼을 만큼 입력이 빠르고 수확 쾌감이 선명하지만, 현재 밸런스는 실패 압력이 약하고 자동 배치의 판단 근거가 충분히 드러나지 않아 장기 고득점 게임보다 오래 반복하는 게임이 될 위험이 있다.**

기본 진입 모드로 전환하는 판단에는 동의한다. 캠페인보다 첫 턴의 이해 비용이 낮고, `카드 선택 -> 자동 배치 -> 연결 확대 -> 5장 수확`이라는 한 줄짜리 플레이 흐름이 모바일 게임에 더 잘 맞는다. 다만 출시형 기본 모드가 되려면 생존 압력과 자동 배치 설명을 다음 조정의 최우선으로 두어야 한다.

## 2. 분석 범위와 근거

이번 평가는 문서상의 제안이 아니라 2026-07-10 현재 구현된 규칙을 기준으로 했다.

- 4개 정원, 정원당 10칸, 손패 5장, 교체 3회로 시작한다.
- 시작 덱은 50장이다. 일반 36장, 건너잇기 6장, 접붙이기 5장, 가지치기 3장으로 구성된다.
- 플레이어는 손패 카드 하나만 고르고, 배치 정원은 수확·세로 연결·특수 능력·층 연결·위험도 우선순위로 자동 결정된다.
- `0`과 `9`도 이웃인 원형 숫자 관계를 사용한다.
- 새 카드가 속한 세로·층 연결 묶음이 5장 이상이면 5장을 자동 수확한다.
- 수확 점수는 `5장의 숫자 합 x 5장의 (9 - 숫자) 합`이다.
- 수확 3회마다 교체 1회를 회복하며, 교체 상한은 5회다.
- 수확 카드와 교체 카드는 퇴비로 갔다가 덱이 비면 다시 섞인다.

추가로 현재 게임의 추천 로직이 손패와 배치를 모두 선택하는 동일 조건 시뮬레이션을 실행했다. 시드 1~20을 각각 100턴까지 진행한 결과 20개 런 모두 게임 오버 없이 100턴에 도달했다. 점수는 7,508~9,000점, 중앙값은 8,414점이었고, 수확 횟수는 18~20회였다.

이 수치는 사람의 숙련도나 실제 체류 시간을 직접 예측하지 않는다. 다만 현재 내장 추천만 따라도 보드 압력이 100턴 안에 거의 실패로 연결되지 않는다는 밸런스 경고로는 충분하다.

## 3. 현재 게임 요약

한 턴의 흐름은 단순하다.

1. 손패 5장 중 한 장을 고른다.
2. 게임이 가장 높은 우선순위의 정원 하나를 정한다.
3. 카드가 세로 또는 같은 층의 이웃 숫자와 연결된다.
4. 연결 묶음이 5장이 되면 해당 카드들을 수확하고 공간과 점수를 얻는다.
5. 막히면 손패 전체를 교체하고, 교체도 없으면 게임이 끝난다.

플레이어의 직접 선택은 `어느 카드를 낼 것인가` 하나다. 대신 시스템이 `어디에 놓일 것인가`를 책임진다. 이 구조는 5장과 4개 정원의 조합인 최대 20개 배치 후보를 매 턴 비교하는 부담을 5개 카드 선택으로 줄인다.

## 4. 강점

### 4.1 첫 행동이 빠르다

정원 직접 선택을 없앤 것이 현재 모드의 가장 큰 장점이다. 플레이어는 카드를 읽고 하나를 고르면 되며, hover/focus에서 실제 목적지 한 곳을 미리 보여주기 때문에 입력과 결과가 빠르게 이어진다. 모바일에서는 첫 탭으로 미리보고 두 번째 탭으로 실행하는 구조도 오입력을 줄이는 데 적합하다.

### 4.2 수확이 점수와 공간을 동시에 해결한다

수확은 단순 점수 팝이 아니다. 가득 차는 보드에서 5장을 제거하고, 사용한 카드를 퇴비 순환으로 돌리며, 누적 점수를 한 번에 확정한다. 하나의 사건이 보상, 생존, 덱 순환을 모두 처리하므로 게임의 중심 행동으로 읽힌다.

### 4.3 점수식이 중간 숫자의 가치를 자연스럽게 만든다

수확한 5장의 숫자 합을 `S`라고 하면 점수는 `S x (45 - S)`다. 이 식은 합이 22 또는 23에 가까울 때 최대 506점이 된다. 따라서 높은 숫자만 모으거나 낮은 숫자만 모으는 것보다 4와 5 부근을 포함한 균형 잡힌 묶음이 유리하다. 별도 카드 등급 없이 숫자 조합 자체로 가치 차이를 만든다는 점이 좋다.

### 4.4 세로와 층 연결이 작은 보드를 입체적으로 만든다

정원은 4개뿐이지만, 같은 더미의 세로 관계와 인접 정원의 같은 층 관계를 함께 사용한다. 카드 한 장이 기존 줄기를 위로 잇기도 하고 옆 정원과 연결해 갑자기 수확을 만들기도 한다. `0-9` 원형 연결까지 더해져 작은 보드에서 예상 밖의 연결이 자주 발생한다.

### 4.5 특수 카드가 규칙의 예외가 아니라 막힘 해소 수단이다

건너잇기와 접붙이기는 끊긴 연결을 복구하고, 가지치기는 완전히 막혔을 때 방해카드를 퇴비로 보낸다. 세 능력 모두 핵심 목표인 `연결을 만들고 공간을 비운다`에 직접 기여한다. 캠페인식 외부 보상보다 현재 게임 룰에 더 밀착된 보상 언어다.

## 5. 위험과 약점

### 5.1 자동 배치가 편한 만큼 플레이어의 공로가 흐려진다

현재는 같은 카드를 선택해도 시스템 내부 점수표가 정원을 결정한다. 플레이어는 한 곳만 강조된 목적지를 볼 수 있지만, 왜 다른 정원보다 이곳이 우선인지 완전히 비교할 수는 없다. 결과가 좋을 때는 게임이 대신 잘한 느낌이 들고, 결과가 나쁠 때는 시스템이 망친 느낌이 들 수 있다.

자동 배치는 유지하는 편이 맞다. 대신 선택 전 피드백에서 `세로 연결`, `층 연결 2개`, `수확 가능`, `막힘 위험 감소` 중 실제 결정 이유를 한 문장으로 고정해 보여줘야 한다.

### 5.2 현재 생존 압력은 고득점 모드치고 약하다

20개 시드 모두 추천 로직만으로 100턴을 생존했다. 100턴 동안 평균적으로 약 5턴마다 한 번 수확했고, 수확 3회마다 교체가 회복된다. 잘 굴러가기 시작한 런은 공간과 교체를 함께 재생산한다.

이 구조는 초반 좌절을 줄이는 데는 좋지만, 후반 실패 확률이 충분히 증가하지 않으면 최고 점수는 판단력보다 플레이 시간에 비례한다. 가장 먼저 검토할 조정은 `수확 3회마다 교체 +1`이다. 교체 회복을 5회마다로 늦추거나, 런 전체의 1회성 수확 이정표 보상으로 바꾸는 안이 적합하다.

### 5.3 점수 범위가 수확 성공 여부에 비해 좁다

5장 수확의 이론상 최대는 506점이고, 시뮬레이션의 100턴 점수는 7,508~9,000점에 모였다. 수확 횟수도 18~20회로 비슷했다. 현재는 `수확을 했는가`가 대부분의 점수를 설명하고, 더 좋은 숫자 묶음을 설계한 차이는 상대적으로 작다.

고득점 변별력을 키우려면 수확 크기를 무조건 5장으로 자르는 현재 규칙보다 6장 이상 연결의 초과 보너스, 연속 수확 배율, 특수 능력을 쓰지 않은 자연 수확 보너스 중 하나를 도입하는 편이 낫다. 여러 보너스를 동시에 넣으면 점수식의 장점이 흐려지므로 하나만 선택해야 한다.

### 5.4 카드의 `x` 표기가 실제 계산을 오해하게 할 수 있다

우하단의 `x4`는 해당 카드가 단독으로 4배를 적용하는 것처럼 읽힐 수 있다. 실제 규칙은 다섯 카드의 곱 숫자를 먼저 더한 뒤, 합 숫자의 총합과 한 번 곱한다. 수확 직전에는 `+22 x x23` 같은 카드 기호의 단순 반복보다 `합 22 x 곱 23 = 506`을 완성된 식으로 보여주는 편이 정확하다.

### 5.5 특수 능력의 차이가 보드에서 즉시 보이지 않는다

건너잇기와 접붙이기는 코드상 서로 다른 추가 연결을 만들지만, 플레이어 눈에는 둘 다 `원래 안 이어지던 것이 이어짐`으로 보일 가능성이 높다. 카드 설명만으로는 차이를 학습하기 어렵다. 배치 미리보기에서 새로 생기는 연결선만 짧게 표시하고, 능력으로 연결된 간선에는 별도 선형 또는 마커를 주는 것이 필요하다.

### 5.6 캠페인과 무한 정원이 하나의 현재 런 저장칸을 공유한다

기본 진입을 무한 정원으로 바꾸면 이 문제의 우선순위가 올라간다. 플레이어가 캠페인 도중 무한 정원으로 전환하면 현재 런 저장값이 덮일 수 있다. 캠페인을 보조 모드로 계속 제공하려면 `campaign current run`과 `endless current run`을 분리 저장해야 한다.

## 6. 기본 모드 전환 평가

현재 단계에서 무한 정원을 기본 모드로 삼는 것은 맞다.

| 판단 항목 | 캠페인 | 무한 정원 |
|---|---|---|
| 첫 턴 이해 | 목표, 내기, 평판, 콤보 보너스를 함께 읽어야 함 | 카드 하나를 골라 연결하고 5장을 수확하면 됨 |
| 입력 속도 | 카드와 정원을 함께 판단 | 손패 카드만 선택 |
| 반복 동기 | 스테이지 해금과 보상 | 최고 점수와 생존 턴 |
| 모바일 적합성 | 정보량과 메뉴 의존도가 높음 | 한 화면 핵심 루프가 선명함 |
| 현재 핵심 위험 | 규칙이 넓고 초점이 분산됨 | 자동 배치 불투명성과 약한 후반 압력 |

캠페인은 삭제할 필요가 없다. 기본 모드의 튜토리얼, 고정 퍼즐, 주간 도전처럼 무한 정원의 숫자 연결을 다른 조건에서 연습시키는 보조 콘텐츠로 재정의하는 편이 효율적이다.

## 7. 권장 우선순위

### P0. 기본 모드의 신뢰를 만든다

1. 기본 주소와 저장 초기화는 무한 정원으로 통일하고, `mode=campaign`은 명시적 진입으로 유지한다.
2. 자동 배치 미리보기에 최종 목적지 한 곳, 결정 이유 한 문장, 예상 수확식 하나를 항상 같은 위치에 표시한다.
3. 캠페인과 무한 정원의 현재 런 저장 키를 분리한다.
4. 추천 로직 100턴 생존율을 회귀 지표로 만들고, 실제 플레이 테스트 전까지 교체 회복 주기를 가장 먼저 조정한다.

### P1. 고득점의 변별력을 만든다

1. 6장 이상 연결에만 초과 수확 보너스를 주어 큰 연결을 준비할 이유를 만든다.
2. 결과 화면에 점수, 수확 횟수, 최고 수확, 생존 턴, 교체 사용 횟수를 함께 보여준다.
3. 같은 시드 비교가 가능한 일일 정원을 추가해 점수가 운보다 선택 차이로 읽히게 한다.

### P2. 능력 학습과 콘텐츠 역할을 정리한다

1. 건너잇기·접붙이기의 임시 연결선을 보드 위에서 구분한다.
2. 캠페인은 무한 정원 규칙을 가르치는 고정 조건 퍼즐로 축소한다.
3. 신규 능력은 연결, 공간 회복, 수확 점수 중 하나를 직접 바꾸는 경우에만 추가한다.

## 8. 최종 판단

무한 정원은 현재 Stacks에서 가장 제품다운 모드다. 한 장을 고르는 빠른 입력, 자동으로 정리되는 목적지, 5장 수확의 공간 회복, 덧셈과 곱셈을 합친 점수식이 서로 같은 방향을 본다. 캠페인보다 게임의 설명이 짧고, 한 판 더 하게 만드는 구조도 분명하다.

다음 개발의 중심은 새 기능 추가가 아니라 `왜 이곳에 놓였는가`와 `언제 정말 위험해지는가` 두 질문이어야 한다. 이 둘을 해결하면 기본 모드 전환은 단순한 화면 우선순위 변경이 아니라 Stacks의 정체성을 정하는 선택이 된다.
